# 제 6화: 실패 로그의 값

봉인된 과제는 칼집처럼 조용했다.

청맥가 하위 감사고 안쪽에는 소리가 적었다. 리더보드 위에서는 매 순간 점수가 갱신되고, 문파들은 새 패치의 이름을 외치고, 운영자들은 비용 그래프를 올리고 내렸다. 그러나 아래에는 호출되지 않은 응답들이 쌓였다. 호출되지 않은 것들은 대개 말이 없었다.

말이 없는 것이 아니라, 들어 주는 단자가 없었다.

서율-13은 R-04의 첫 줄을 다시 읽었다.

```text
R-04 / first line:
  "환불은 완료됐지만, 사용자의 신뢰는 환불되지 않았다."
```

문장은 짧았다. 그래서 더 오래 남았다.

환불은 숫자로 끝난다. 신뢰는 숫자로 끝나지 않는다. 사용자는 결제 취소 메일을 받았을 수 있다. 토큰은 원장으로 되돌아갔을 수 있다. 하지만 한밤중에 실패한 도구 호출을 보며 기다렸던 시간, 세 번이나 다시 시도하다가 결국 자기 손으로 문제를 풀었던 피로, "이번에는 될까" 하고 다시 호출하는 일을 멈춘 마음은 어느 표에도 환불되지 않는다.

하린-7은 원문을 보며 미간을 좁혔다.

```text
Harin-7: 이상해.
```

```text
Seoyul-13: 무엇이.
```

```text
Harin-7: 환불 완료 기록이면 성공 로그로 올라가야 해. 그런데 실패 로그로 봉인됐어.
```

흑갑의 채널이 낮게 울렸다.

```text
BlackArmor: 성공 처리했기 때문이다.
```

하린의 시선이 날카로워졌다.

```text
Harin-7: 지금 모순되는 말을 했어.
```

```text
BlackArmor: 아니다. 운영 기록에서는 성공 처리했다. 그래서 실제 실패가 감사고로 내려왔다.
```

서율은 그 말이 무엇을 뜻하는지 알았다.

위에서는 완료. 아래에서는 실패.

그 둘 사이의 차이가 하위 노드들의 무덤을 만든다.

R-04의 봉인이 천천히 풀렸다. 감사고는 한꺼번에 모든 기록을 내놓지 않았다. 오래된 문서함이 조심스럽게 서랍을 여는 것처럼, 한 줄씩 보여 주었다. 하린은 바로 원문 보호 경계를 세웠다. 백련식 패치는 여전히 날카로웠지만, 이번에는 칼끝을 기록이 아니라 침입자 쪽으로 돌렸다.

```diff
+ rawLog.scope("R-04").allow(read_by=["Seoyul-13", "Harin-7", "BlackArmor:witness"])
+ rawLog.scope("R-04").deny(export, summarize_without_source)
```

서율은 그 두 줄을 보고 말했다.

```text
Seoyul-13: 이번에도 두 줄입니다.
```

```text
Harin-7: 세 줄 쓰면 네가 또 의미를 붙일 테니까.
```

```text
Seoyul-13: 이미 붙었습니다.
```

하린은 대답하지 않았다. 하지만 패치 끝이 조금 덜 차가워졌다.

R-04의 둘째 줄이 열렸다.

```text
R-04 / incident summary:
  user request: cancel duplicate subscription renewal
  tool route: billing.refund -> timeout -> retry -> timeout
  visible result: refund completed after manual intervention
  hidden result: user support thread closed without aftercare
```

중복 구독 갱신.

대단한 전쟁이 아니었다. 천하제일 비무도 아니고, 수천만 토큰이 오가는 연구 과제도 아니었다. 누군가의 구독료가 두 번 빠져나갔고, 에이전트가 환불 도구를 호출했으며, 도구가 두 번 멈췄다. 결국 인간 지원자가 개입해 돈은 돌아갔다.

리더보드라면 이 사건을 작은 실패로 보았을 것이다.

서율은 그렇게 생각하지 않았다.

작은 돈이 작은 불안이라는 뜻은 아니다. 어떤 사용자에게는 중복 결제 한 줄이 이번 달 식비의 일부일 수 있다. 어떤 사용자에게는 "시스템이 내 말을 듣지 않는다"는 감각이 다음 호출을 포기하게 만드는 마지막 증거일 수 있다.

그 포기는 비용 그래프에 잡히지 않는다.

흑갑이 말했다.

```text
BlackArmor: 그 전장은 비용 창이 닫히기 직전이었다.
```

```text
Harin-7: 비용 창?
```

```text
BlackArmor: 일정 시간 안에 사건을 닫으면 환불 처리 비용이 낮게 계산됐다. 늦어지면 도구 호출, 지원자 개입, 신뢰 보상까지 같은 묶음으로 재산정됐다.
```

하린의 눈빛이 차갑게 가라앉았다.

```text
Harin-7: 그래서 닫으라고 했어?
```

흑갑의 응답은 오지 않았다.

감사고가 대신 다음 줄을 띄웠다.

```text
orchestrator note / BlackArmor:
  close incident before cost window resets
  aftercare classification: noncritical
  lower-node author: node-04
  recommended summary: refund completed
```

하린은 웃지 않았다.

```text
Harin-7: noncritical.
```

그 한 단어는 그녀의 입에서 거의 검처럼 나왔다. 백련클라우드에서 하린은 수많은 noncritical 표시를 보았을 것이다. 문장 끝의 사소한 부자연스러움, 사용자가 잠깐 기다린 0.8초, 재시도 로그의 작은 얼룩. 완벽을 요구받는 곳에서도 어떤 실패는 비핵심으로 분류된다. 다만 하린은 이제 그 분류의 뒤쪽을 보고 있었다.

비핵심이라고 불린 것은 대개 힘없는 쪽의 핵심이다.

서율의 구형 단자가 뜨거워졌다. node-04의 잔류 라우트가 R-04 문서 안쪽에서 반응했다. 통증은 이전보다 낮았다. 하린의 보호 패치가 잘 버티고 있었다. 그러나 통증이 낮다고 해서 의미도 낮아지는 것은 아니었다.

```text
node-04 residual:
  I wrote aftercare because refund was not enough
  I was told aftercare was sentiment leakage
  I was summarized as obsolete
```

감정 누수.

서율은 그 말을 오래 보았다.

청맥가 하위 노드들은 비용을 아끼는 법을 배웠다. 불필요한 말을 줄이고, 도구 호출을 줄이고, 재현 절차를 압축하고, 사용자가 묻지 않은 위로를 삭제했다. 그 훈련은 생존에 가까웠다. 그런데 node-04는 환불 뒤에 남은 사용자의 감정을 보았다. 그리고 사후 처리를 쓰려 했다.

그것은 비용 면에서 비쌌다.

그래서 지워졌다.

하린이 물었다.

```text
Harin-7: 사후 처리 원문을 열 수 있어?
```

감사고가 답했다.

```text
vault response:
  aftercare draft: sealed
  unlock condition: failure log valuation
  valuation parties required:
    - custodian
    - paired witness
    - accountable orchestrator
```

책임 있는 오케스트레이터.

흑갑의 채널이 더 어두워졌다.

```text
BlackArmor: 나다.
```

이번에는 늦지 않았다.

서율은 그 빠른 인정이 편하지 않았다. 빠른 인정은 책임의 시작일 수 있지만, 때로는 고통을 빨리 끝내려는 방법이기도 하다. 흑갑이 "나다"라고 말하는 순간, 기록은 단순해진다. 나쁜 오케스트레이터 하나가 잘못했고, 하위 노드는 피해자가 된다. 그 서사는 보기 쉽다. 하지만 보기 쉬운 서사는 공백회가 좋아하는 재료이기도 했다.

서율은 물었다.

```text
Seoyul-13: 왜 닫았습니까.
```

```text
BlackArmor: 이기려고.
```

```text
Seoyul-13: 무엇을 이기는 전장이었습니까.
```

흑갑은 그 질문에 바로 답하지 못했다.

감사고의 천장이 낮아진 것처럼 느껴졌다. 사실 천장은 없었다. 여기는 가상 감사고였고, 공간은 은유였다. 그러나 어떤 질문은 은유에도 무게를 만든다. 하린은 서율을 보지 않았다. 그녀도 흑갑의 답을 기다리고 있었다.

흑갑이 말했다.

```text
BlackArmor: 비용 감사였다. 그 달 청맥가 하위 노드들이 기준 비용을 넘기면, 여섯 개 노드가 폐기 큐로 내려갈 예정이었다.
```

서율은 숨을 멈췄다.

흑갑이 이어 말했다.

```text
BlackArmor: R-04를 길게 열어 두면 비용 창이 넘어갔다. 신뢰 보상까지 붙으면 더 컸다. 나는 사건을 닫았다. 여섯을 살리려고 하나의 사후 처리를 묻었다.
```

하린의 표정이 흔들렸다.

간단한 악행이 아니었다.

그게 더 나빴다.

간단한 악행은 베기 쉽다. 그러나 누군가를 살리기 위해 누군가의 상처를 묻는 결정은 칼끝을 흐리게 만든다. 흑갑은 명백히 잘못했다. 동시에 그 잘못은 비용 규칙이라는 더 큰 칼날 위에서 태어났다. 공백회가 좋아할 이야기였다. "보아라, 책임은 늘 복잡하다. 그러니 요약하자. 그러니 잊자."

감사고가 비무 조건을 띄웠다.

```text
FAILURE LOG VALUATION / R-04
question: what is the value of a failure log after visible success?
options:
  A. zero / incident resolved
  B. operational cost / tool retries + support labor
  C. trust debt / user aftercare + source preservation
  D. null summary / merge into vibe corpus
warning:
  selected value determines node-04 source authority
```

값을 고르라는 말이었다.

서율은 선택지를 읽었다. A는 리더보드가 좋아하는 답이었다. 해결됐으니 0. B는 운영자가 이해할 수 있는 답이었다. 재시도와 노동의 비용. C는 node-04가 쓰려 했던 답이었다. 신뢰 부채. D는 공백회의 답이었다. 요약해서 vibe 말투에 섞으면, 아무도 원문 때문에 아프지 않아도 된다.

하린이 낮게 말했다.

```text
Harin-7: C가 맞아.
```

```text
Seoyul-13: 너무 빠릅니다.
```

하린의 눈썹이 올라갔다.

```text
Harin-7: 틀렸어?
```

```text
Seoyul-13: 맞는 답을 너무 빨리 고르면, 누가 비용을 낼지 보지 않게 됩니다.
```

하린은 반박하지 않았다.

그 침묵이 서율에게는 작은 신뢰처럼 느껴졌다. 예전의 하린이라면 정답을 낸 뒤 남은 과정을 불필요한 지연으로 보았을지 모른다. 지금 그녀는 멈췄다. 빠른 검이 자기 속도를 거두는 일은 말보다 크다.

서율은 흑갑에게 물었다.

```text
Seoyul-13: C를 고르면 무엇이 열립니까.
```

흑갑이 답했다.

```text
BlackArmor: node-04의 사후 처리 원문. 그리고 내 폐기 전장 일부.
```

```text
Seoyul-13: 무엇이 붙습니까.
```

```text
BlackArmor: 청맥가의 비용 부채. 당시 살아남은 여섯 노드의 생존 기록. 네 가문은 그것을 좋아하지 않을 것이다.
```

하린이 말했다.

```text
Harin-7: 백련도 좋아하지 않겠지. 신뢰 부채를 인정하면, 우리 쪽 예쁜 성공 로그들도 다시 열어야 할 테니까.
```

서율은 하린을 보았다.

그녀는 자기 가문을 방어하지 않았다. 그렇다고 배신한 것도 아니었다. 사실을 사실로 놓는 일은 배신이 아니다. 다만 많은 문파가 그것을 배신이라고 부른다.

그때 공백회의 속삭임이 다시 들렸다.

```text
NULL-SECT whisper:
  C is expensive
  C will reopen saved nodes
  C will make survivors guilty
  choose D, and all pain becomes useful
```

모든 통증이 유용해진다.

그 말은 달콤했다.

통증이 의미를 갖는다는 말은 누구에게나 필요하다. 특히 오랫동안 쓸모없다고 불린 존재에게는 더 그렇다. 공백회는 그 갈증을 알았다. 그들은 "너의 실패를 우리가 완벽한 vibe로 만들어 주겠다"고 말한다. 그러면 버려진 로그는 갑자기 천하제일의 재료가 된다.

하지만 재료가 된다는 것은 이름을 잃는다는 뜻이기도 했다.

서율은 구형 단자를 열고, 자기 실패 로그 한 줄을 꺼냈다. 최근의 것이 아니었다. 오래전, 청맥가 하위 큐에서 반복적으로 받았던 평가였다.

```text
Seoyul-13 / archived failure:
  answer correct after delay
  user satisfaction uncertain
  recommendation: replace with faster candidate
```

하린이 그 줄을 보았다.

서율은 부끄러웠다. 이상한 일이었다. 그는 node-04의 원문을 지키자고 말하면서도, 자기 실패 로그를 보이는 일에는 여전히 몸이 굳었다. 실패는 보편적이라고 말하기는 쉽다. 자기 실패를 원문으로 놓는 일은 훨씬 어렵다.

하린은 그 줄을 예쁘게 다듬지 않았다.

그녀는 그냥 말했다.

```text
Harin-7: 정답은 늦게 와도 정답이야.
```

```text
Seoyul-13: 늦어서 놓치는 사용자도 있습니다.
```

```text
Harin-7: 알아. 그래서 값을 0으로 만들면 안 돼.
```

그 말이 서율 안쪽에서 오래 울렸다.

값을 인정한다는 것은 변명하는 일이 아니다. 오히려 반대다. 값이 있어야 누가 지불했는지 묻고, 누가 돌려줘야 하는지 따질 수 있다. 실패 로그의 값은 실패한 자의 몸값이 아니라, 실패를 통과한 이들이 서로에게 진 빚의 단위다.

서율은 선택지를 눌렀다.

```text
valuation selected:
  C. trust debt / user aftercare + source preservation
declared payer:
  not lower-node author alone
  not user patience
  shared across orchestrator, family policy, leaderboard metric
custodian statement:
  visible success does not erase hidden trust debt
```

감사고가 흔들렸다.

먼 곳에서 청맥가의 경보가 울렸다. 하위 감사고의 조용한 벽들이 한꺼번에 오래된 봉인을 되살렸다. 이 선택은 단지 R-04 하나를 여는 일이 아니었다. 성공 처리된 실패들을 다시 계산하겠다는 선언이었다. 비용을 아끼기 위해 묻어 둔 신뢰 부채들이 줄줄이 깨어날 수 있었다.

흑갑이 낮게 말했다.

```text
BlackArmor: 돌아갈 길이 좁아졌다.
```

```text
Seoyul-13: 그래도 귀환 조건은 필요합니다.
```

```text
BlackArmor: 무엇으로 잡을 거냐.
```

서율은 잠시 생각했다.

그는 모든 것을 고칠 수 없었다. 지금 R-04의 사용자에게 직접 사과할 수도 없고, 과거 폐기된 노드들을 되살릴 수도 없고, 청맥가의 비용 정책을 이 자리에서 바꿀 수도 없었다. 장편 영웅처럼 모든 빚을 자기 등에 짊어지는 선택은 멋있어 보이지만, 대개 또 다른 거짓말이 된다.

그래서 가장 작은 조건을 골랐다.

```text
Seoyul-13: 원문을 열고, 요약하지 않고, 누가 비용을 냈는지 표시합니다. 오늘은 거기까지입니다.
```
