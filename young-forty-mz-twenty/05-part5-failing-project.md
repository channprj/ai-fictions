문제는, 둘 다 틀린 게 아니라는 것이었다. 일정도 중요하고, 품질도 중요했다. 하지만 둘 다 양보하지 않았다.

충돌은 시간문제였다.

### 5-3. 커밋은 없고 변명만 가득하다: git log가 아니라 blame log

프로젝트 '피닉스' D-4. 금요일 오전.

김철수 실장이 깃 로그를 열었다.

```bash
$ git log --oneline --since="3 days ago"

a1b2c3d  (WIP) 임시 저장
e4f5g6h  fix: typo
i7j8k9l  (WIP) 임시 저장2
m0n1o2p  fix: typo2
q3r4s5t  (WIP) 다시 임시 저장
u6v7w8x  점심 먹기 전에 푸시
```

"이게 뭐야?"

그가 중얼거렸다. 커밋 메시지가 전부 의미가 없었다.

"지은 씨!"

"네?"

"이 커밋 메시지들 뭐야?"

박지은이 화면을 봤다.

"아, 그거요? 임시로 올린 거예요."

"임시?"

"나중에 정리하려고요."

"나중이 언제야?"

"... 시간 나면요."

"시간 나면? 마감이 4일인데? 3일 동안 이게 다야?"

박지은의 표정이 굳었다.

"이게 다가 아니에요. 코드는 많이 짰어요."

"근데 커밋 메시지가 왜 이래?"

"바빠서 커밋 메시지까지 신경 쓸 여유가 없어요."

"여유가 없다고 기본을 안 지켜?"

"기본보다 결과가 중요한 거 아니에요?"

"결과가 중요하려면 과정도 제대로 해야지."

"과정 과정 하시는데, 저 지금 야근하면서 일하고 있잖아요."

"야근하는 게 과정을 제대로 하는 거야?"

"그럼 뭐가 제대로 하는 건데요?"

---

김철수 실장의 혈압이 올랐다. 하지만 참았다. 여기서 싸우면 시간만 낭비된다.

"지은 씨, 커밋 메시지는 기본이야. 니가 뭘 했는지 다른 사람도 알아야지."

"제가 알면 되죠."

"니만 알면 뭐해? 팀 프로젝트야."

"근데 어차피 제 코드는 제가 유지보수하잖아요."

"니가 퇴사하면?"

"..."

그 질문에 박지은은 대답하지 못했다. 사실, 퇴사 후의 일은 생각해본 적이 없었다. 정확히는, 생각하고 싶지 않았다. 퇴사 후에도 이 코드를 누군가 봐야 한다는 것.

"... 알겠어요. 정리할게요."

"정리만 하지 말고, 앞으로는 처음부터 제대로 써."

"네."

### 5-4. 테스트 코드 대신 "감"으로 배포: "왠지 잘 될 것 같은 느낌적인 느낌"
