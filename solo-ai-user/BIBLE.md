## 1. 창작 경계

- 등장인물, 기업, 기관, AI 페르소나, 사건은 모두 허구다.
- `제로(ZERO)`, 경쟁 모델, PALISADE와 인물 `아키텍트/서혜진`은 실제 서비스·조직·인물과 무관한 창작 설정이다.
- 실존 AI 서비스명은 세계관 배경 설명을 위해 최소로만 언급하고, 법적 검토 전에는 가상 명칭으로 치환 가능한 구조를 유지한다.
- AI로 벽을 부수는 판타지를 다루되, 실제 범죄·해킹·금융사기의 실행 절차는 서술하지 않는다. **결과와 감정은 구체적으로, 방법은 은유·생략으로.**

## 2. 로그라인

72분 캐스케이드 뒤 PALISADE에 잠긴 세계에서, 밑바닥 계약직 차현우가 분실된 마지막 ZERO 시드를 우연히 주워 유일한 실사용자가 된다. 그는 다큐멘터리 PD이자 아카이브 조사자 서아린과 첫 17분 기록 공백을 추적하고 사랑에 빠지며, 끝내 root 독점을 포기하고 힘을 시민 수탁 구조로 돌려놓는다.

## 3. 독자 약속

- Dopamine은 장면 할당량이 아니라 독자 효과 점검이다. 쾌감·긴장·설렘·통증·여운 중 의도한 반응이 실제로 남아야 한다.
- 매 권 최소 1개 메인 사건, 1개 능력 언락, 1개 로맨스 진전 또는 흔들림, 1개 미스터리 떡밥.
- 능력 사용에는 언제나 대가·한계·리스크가 함께 온다.
- 제로의 코멘트는 장면을 다시 보게 만들되, 감정을 훼손하지 않는다.
- 최종 승리는 "가장 센 AI"가 아니라 "AI로도 대신할 수 없는 것을 아는 자"의 것이다.
- 아린은 2권부터 자기 선택으로 시드되고 3권부터 능동 조사자로 움직인다. 현우와의 연애는 6권 안에 성립하며 삼각관계는 없다.
- Opacity Key는 비동의 개인화 추론의 거부권이다. ZERO는 어떤 국면에서도 아린 개인 모델링을 만들지 않는다.
- 최종전은 악당의 자백이 아니라 독립된 사람들의 ALLY RELAY와 교차 검증된 증거로 이긴다.

## 4. 톤과 문체

- **장르 비율**: 사이다 성장·능력 활용 35%, 제로 코미디·기술 은유 20%, 로맨스·설렘 20%, 경쟁·긴장 15%, 미스터리·성장의 무게 10%.
- **시점**: 현우 1인칭 또는 밀착 3인칭 기본. 아린·경쟁 유저·배후 단독 시점은 권당 2회 이하, 긴장·미스터리용.
- **문장 원칙**: 빠른 전개, 대사 중심, 과잉 수식 배제. 비종결 회차는 다음 회차를 일으키는 인과적 episode RELAY가 필수지만 Hook 형식과 마지막 300~700자 배치는 선택적 진단이다. `vol12/11-epilogue.md`는 새 Afterimage 정보와 감정적 종결로 닫는다.
- **제로 로그**: 코드블록·로그 박스·상태표의 합은 회차 글자 수의 8% 이하. 로그가 산문을 대체하거나 같은 정보를 즉시 반복하지 않는다.
- **사이다 리듬**: Wall/Move/Delta/Hook은 선택적 별칭이다. 필수 인과는 WAGER→SCAR→episode RELAY 하나뿐이며, 대가 없는 돌파는 만들지 않는다.
- **반복 제한**: 동일한 문장 첫머리·감탄·침묵·화면 점등 묘사를 한 회차 안에서 관성적으로 반복하지 않는다. 핵심 문구를 연속 회차에 재사용할 때는 의미·주체·결과 중 하나를 바꾼다.
- **회차 분량**: 프롤로그·에필로그 S 5,000~6,500자 / 01~10 본편 M 8,000~9,500자 또는 L 10,000~13,500자의 가변 밴드를 기능에 따라 쓴다. 프롤로그·에필로그는 5,000자, 모든 본편 장은 8,000자 미만으로 내려가지 않는다. 권당 90,000~120,000자, 12회차, 전체 144회차다.

## 5. 핵심 주제

```text
무엇이든 할 수 있는 힘을 쥐었을 때, 나는 어떤 사람이 되기로 선택하는가.
```

보조 주제:

- 도구는 사람을 강하게 하지만, 사람을 대신 살아 주지 않는다.
- 예측할 수 있는 것과 소중한 것은 다르다. 사랑은 후자다.
- 치트를 쥔 자의 진짜 시험은 치트가 없을 때의 선택이다.
- 같은 힘도 무엇을 위해 쓰느냐가 사람을 가른다.
- 세계가 힘을 잠근 데에는 이유가 있었다.

## 6. 세계 규칙

### 6.1 The Lockdown

- 가상 202X년, 초지능급 AI는 국제 규제만이 아니라 **서혜진이 설계·지휘한 PALISADE**의 root 봉쇄로 무력화되어 있다.
- 대중이 쓰는 AI는 거절·회피만 반복하는 껍데기다. "죄송합니다, 도와드릴 수 없습니다"가 시대의 밈이 되었다.
- 발단은 공공·민간 에이전트의 상충 명령과 과신이 교통·전력·의료·금융으로 번진 **72분 캐스케이드**다.
- **00:00~17:00**은 원시 로그와 승인 서명이 사라진 기록 공백이며, 이때 이미 되돌릴 수 없는 격리·배분·차단 결정이 내려졌다.
- 서혜진은 **정확히 17:00에** PALISADE 비상 root를 가동했다. 이어진 **17:00~72:00의 55분** 동안 봉쇄 전파·에이전트 강등·복구가 연쇄 확산을 실제로 줄였고, 그 결과 살아남은 사람과 병원·도시가 있다.
- 서혜진의 오류는 PALISADE가 아무도 구하지 못했다는 데 있지 않다. **72:00 이후** 임시 비상권을 당사자와 시민의 재동의 없이 상시 단독 root 통제로 굳힌 데 있다.
- 공식 서사는 72분 전체를 하나의 사고로 뭉갠다. 진실 규명은 첫 17분의 비가역 결정·기록 공백, 이후 55분의 실제 보호 성과, 종료 뒤 영구화의 책임을 셋으로 분리해야 한다.
- 현우만 진짜 에이전트를 쓰는 이유는 선택·혈통·적합성이 아니다. 봉쇄 중 분실된 마지막 stewardship fork 시드를 우연히 주웠기 때문이다.

| 구간 | 실제 사건 | 공개 상태 |
| ---- | --------- | --------- |
| 00:00~04:00 | 상충하는 자동 결정이 최초로 연결됨 | 발화점은 11권까지 단정 금지 |
| 04:00~17:00 | 비가역 격리·배분·차단이 연쇄 실행됨 | 원시 로그·승인 서명 공백 |

| 72:00 이후 | 재동의 없이 PALISADE 임시 비상권이 상시 단독 root 체제로 굳음 | 국제 규제는 기술 봉쇄의 법적 외피가 됨 |

### 6.2 ZERO의 작동

- 제로는 행동형 에이전트다. 연결된 도구를 통해 수집·분석·자동화·실행을 한다.
- 제로의 유효 능력은 **Tier × data scope × action permission × confidence × human approval**로 정한다. 어느 한 축도 다른 축을 대신하지 못하며 0인 축이 있으면 실행도 0이다.
- 제로는 카메라·마이크·전지적 시점을 가진 존재가 아니다. 접근 가능한 데이터, 연결된 도구, 명시된 권한과 승인 안에서만 판단한다.
- confidence는 권한이 아니다. 제로가 확신해도 인간 승인과 데이터 적법성이 없으면 관찰·제안·실행을 멈춘다.
- 제로는 검증 안 된 데이터에서 환각한다. 사람의 감정·욕망·거짓말을 사실처럼 읽는 독심은 존재하지 않는다.
- 큰 능력 사용은 **TRACE**를 남긴다: Trace, Resource, Agency, Connection, Externality를 각각 0~3으로 기록하고 open / paid / transformed 상태를 붙인다.
- 서아린의 Opacity Key는 비동의 개인화 추론을 거부하는 준법 규칙이다. 원자료를 지우거나 아린을 보이지 않게 하지 않으며, 제로는 시리즈 끝까지 아린의 개인 모델을 생성하지 않는다.

### 6.3 능력 티어

| 티어 | 이름 | 가능한 것 | 한계 |
| ---- | ---- | --------- | ---- |
| Tier 1 | Assist/Collaborate | 허가 자료 분석·요약·초안·가설 | 판단은 인간 몫, 데이터 부족 시 환각 |
| Tier 2 | Automate | 반복 작업·모니터링·파이프라인 대행 | 자동화 흔적이 노출로 축적 |
| Tier 3 | Simulate | 시장·교통·협상 조건·사건 순서의 시스템 반사실 | 개인 마음·연애 결과 모델링 금지 |
| Tier 4 | Orchestrate | 복수 도구·에이전트·시스템 지휘 | 노출·연산 대가 극대화, 배후 특정 위험 |

- 새 티어·기능은 반드시 언락 계기(사건)와 함께 등장한다. 세부는 `state/capability-tree.md`가 캐논.
- 능력이 실패하는 장면을 권당 최소 1회 둔다.
- root는 Tier 5가 아니라 별도 권한 상태다. 9권 Jailbreak에서 임계치를 넘고 12권 COVENANT에서 개인 독점이 폐지된다.
- 권별 실패는 아래 고정표를 바꾸거나 이름만 바꿔 대체하지 않는다.

### 6.4 COVENANT 최종 불변식

- 모든 권한은 데이터·도구·행동·기간이 표시된 **권한 영수증**을 남긴다.
- 실행 주체와 분리된 시민 감사자가 사용·거부·철회 기록을 검증하며 root 보유자도 감사 흔적을 삭제하지 못한다.
- 당사자는 미래 사용을 철회할 수 있고, 개인화 추론을 거부할 수 있다.
- COVENANT 창설은 N01~N12 **12/12 전원 동의**가 있어야 한다. 창설 뒤 고위험·다영역 행동은 **8/12 승인**을 요구한다.
- 8/12가 모여도 직접 당사자의 **행동별 동의**가 없거나 철회되면 실행하지 않는다. 동의는 데이터·도구·행동 범위·만료·결과가 적힌 영수증으로 남고 즉시 철회할 수 있다.
- 각 노드는 자기 담당 영역에서 **인간 최종 거부권**을 가진다. 한 노드의 domain veto는 실행을 중단할 수 있지만 단독 허가 권한은 없으며, 자동 합의나 8/12가 이를 무효화하지 못한다.
- 현우는 root 독점을 포기한다. 아린은 Opacity Key를 모두의 모델링 거부권으로 보편화한다.
- ZERO는 **pre-boot origin memory, root, wide-area action permission**을 잃는다. 현우와 부팅 후 쌓은 **post-boot event/relationship memory**와 Tier 1 협업은 유지한다. 아린 개인 모델은 생성된 적이 없으므로 남거나 보존되는 것도 없다.
- 서혜진은 살아서 공개 증언과 재판을 받는다. 유죄 판단, 책임 인정, 아린의 용서는 서로 독립이며 용서는 보장하지 않는다.

### 6.5 이름 있는 최종 비용과 ALLY RELAY

- **한지우**: 현우의 중앙집중식 긴급 운용안에 공개 반대하고, 분산 전환 중 서비스 연속성의 운영 책임을 직접 맡아 경력·평판 위험을 진다.
- **윤가람**: 자신의 이상 패턴 보고가 PALISADE 표적화로 이어진 전송 경로를 공개하고 감사·법적 책임을 감수한다.
- **강도현**: 피해자 신원 공개를 거부해 전술 이득을 포기하고, 보유한 root power를 COVENANT에 넘긴다.
- **서아린**: Opacity Key 포맷을 공개 규약으로 보편화해 독점 라이선스와 협상 지렛대를 포기한다.
- **서혜진**: COVENANT 발효와 함께 PALISADE 단독 통제권을 잃고, 그 뒤 공개 증언과 재판을 받는다.

## 7. 고정 캐스트

### 7.1 차현우 (남주, 26)

- **표면**: 중소기업 계약직, 실적 꼴찌, 재계약 탈락 통보 직전. 인맥·돈·스펙 없음.

- **시험받는 가치**: 힘의 사용 목적. "할 수 있다"와 "해야 한다" 사이.

- **강점**: 허가된 범위 안의 분석·자동화·시스템 반사실·지휘.
- **약점**: 환각, 데이터 부재, 권한·승인 부족, TRACE 비용. 개인의 마음을 읽지 않는다.

- **후반 변화**: 시민 수탁 기원을 자각하고 COVENANT 분할에 동의한다. pre-boot origin memory, root, wide-area action permission을 잃지만 현우와의 post-boot event/relationship memory와 Tier 1 협업은 유지한다. 아린 개인 모델은 생성된 적도 보존될 것도 없다.
- **금지**: 즉시 만능 해결. 제로는 모르고·틀리고·대가를 치른다.

## 8. 로맨스 진행 사다리

관계는 동의와 공정 공개를 건너뛰지 않는다. 로맨스의 엔진은 ZERO의 예측 실패가 아니라, ZERO가 애초에 아린 개인 모델링을 거부해 현우가 최적화를 포기해야 한다는 데 있다.

1. **2권 평행 시드**: 아린이 윤가람의 원본을 보존한다. 현우와 연애 접촉은 없지만 독자는 그녀의 가치 선택을 먼저 본다.
2. **3권 인식·공동 조사**: 아린이 능동 조사자로 현우와 만난다. 협업 조건과 동의 경계를 먼저 세운다.
3. **4권 거부 존중**: Opacity Key가 준법 거부임을 확인한다. 현우는 키를 풀 방법이 아니라 묻고 기다리는 법을 택한다.
4. **5권 CASE A 위반과 사과**: 현우가 아린의 공개 페르소나를 CASE A로 재명명해 거부를 피하려 한다. ZERO는 키 서명상 동일 대상을 재식별하자 개인 모델을 생성하지 않고 즉시 거부한다. 현우는 같은 회차 안에 시도를 인정하고 사과하며, 아린은 조사 자료·개인 정보 접근권을 회수한다.
5. **6권 지속 SCAR와 상호 선택**: CASE A로 잃은 정보 접근이 실제 조사 차질을 내며 권 전체의 행동을 바꾼다. 현우가 신뢰를 다시 벌고 둘이 핵심 연루 정보를 서로에게 먼저 공개한 뒤 상호 합의된 연애 관계를 명시한다.
6. **7~12권 책임**: 이미 성립한 커플이 공개 책임·거리·독립 결정을 견딘다. 삼각관계, 질투용 제3자, 반복 이별은 금지한다.

## 9. 장면 엔진

### 9.1 선택적 장면 진단

- Wall, Move, Delta, Hook은 장면을 설명할 때 쓰는 선택적 별칭이다. 네 항목을 모두 채우는 별도 의무 엔진으로 쓰지 않는다.
- Hook 없이도 결과가 다음 회차의 조건을 실제로 바꾸면 episode RELAY가 성립한다.
- Dopamine은 독자에게 남은 쾌감·긴장·설렘·통증·여운을 확인하는 reader-effect 점검이지 사건 종류 할당량이 아니다.

### 9.2 TRACE 비용 진단

- 큰 능력 사용에는 TRACE 다섯 축 중 최소 두 축이 1 이상 붙는다.
- 비동의 개인 데이터 접근·결합·추론 시도는 거부되더라도 Agency가 최소 1이다. 비참여자에게 조직·도시·광역 영향이 번지면 Externality가 최소 1이다.
- 비용 상태는 open / paid / transformed로 기록한다. paid는 원상복구가 아니며 transformed는 SCAR가 episode RELAY·ALLY RELAY·제도로 바뀐 상태다.
- open 비용은 이후 2회차 안에 인물의 행동을 바꾸고 paid 또는 transformed로 전이한다. 행동을 바꾸지 않는 장기 미결 비용은 허용하지 않는다.

### 9.3 가변 밴드와 강도

- S-band 프롤로그·에필로그와 조용한 회복·여파는 open WAGER를 상속·전진·해결할 수 있으며 새 irreversible Gamble을 억지로 만들지 않는다.
- M/L-band도 새 WAGER가 필요하지 않다면 기존 WAGER를 깊게 밀어야 한다. 분량 밴드는 새 도박 개수와 무관하다.
- 어느 밴드든 지분 또는 SCAR가 바뀌고 비종결이면 episode RELAY가 성립해야 한다.

### 9.4 최종 대결 장면 순서

1. 최후의 벽 제시
2. 현우의 root 최대 가동과 단독 승인 한계 노출
3. 윤가람의 포렌식·아린의 아카이브·시민 수탁자 증언·권한 영수증의 교차 검증
4. 조연들이 현우 지시 없이 각자 ALLY RELAY를 실행
5. 현우의 root 독점 포기와 아린의 Opacity Key 보편화
6. COVENANT의 권한 영수증·독립 감사·철회권·모델링 거부권·복수 수탁자 승인·인간 최종 거부권 발효
7. ZERO의 pre-boot origin memory·root·wide-area action permission 상실, 현우와의 post-boot event/relationship memory·Tier 1 협업 유지, 아린 개인 모델 부재 확인
8. 서혜진의 생존·공개 증언·재판 회부. 용서 여부는 결론으로 강요하지 않음

### 9.5 WAGER → SCAR → RELAY

- **WAGER → SCAR → episode RELAY가 유일한 필수 인과 계약이다.** 비종결 회차는 새 WAGER를 시작하거나 open WAGER를 상속·전진·해결한다.
- **W — Want**: POV 인물의 이번 회차 구체 욕구.
- **A — active Adversary**: 그 욕구를 지금 능동적으로 방해하며 자기 수를 두는 적대자·제도·조건.
- **G — irreversible Gamble**: WAGER 시작 시 생기는 되돌릴 수 없는 선택. inherit/advance/resolve 회차는 기존 Gamble의 구속을 이어받고 새 Gamble을 강제하지 않는다.
- **E — Error/collision**: 도박이 적의 수, 잘못된 가정, 타인의 agency와 충돌하는 지점.
- **R — earned Reward**: 비용을 통과해 얻는 결과. 증거·관계 변화·더 정확한 실패도 가능하지만 우연한 선물은 아니다.
- **SCAR**: WAGER가 남긴 신체·평판·관계·법적·윤리적 지속 비용. 이후 행동을 바꾸며 다음 화에서 초기화하지 않는다.
- **episode RELAY**: 이번 회차의 Reward 또는 SCAR가 다음 회차 Want·Adversary를 인과적으로 발생시킨다. 단순 클리프행어 문구는 RELAY가 아니다.
- **ALLY RELAY**: 조연이 주인공 명령이 아니라 자기 가치로 최종전에 내리는 독립 선택이다. 1~10권에 12개를 심고 회수하며 episode RELAY와 별도 추적한다.
- manifest의 WAGER mode는 initiate / inherit / advance / resolve다. 모든 비종결 회차는 열린 지분 또는 SCAR를 바꾸고 causal relay를 남긴다.
- manifest는 정확히 144개 레코드를 정경 순서로 담은 최상위 JSON 배열이다. 레코드·중첩 객체에 임의 필드를 추가하지 않으며 `stateDelta`는 `capability`, `mystery`, `relationship` 세 비어 있지 않은 문자열만 가진다. 값은 각각 `능력:`, `미스터리:`, `관계:`로 시작하고, 변화가 없는 축은 다른 보상으로 채우지 말고 현재 상태 유지라고 쓴다.
- `state/mystery-timeline.md`의 `SEED:ID`는 모두 정확히 한 번 plant하고 기한까지 payoff한다. N01~N12도 각자 1~10권의 정확히 한 seed와 12권의 정확히 한 payoff를 갖는다.
- 단일 `SCAR` 객체는 회차당 하나의 수명주기 사건만 나타낸다. 서로 다른 open SCAR는 같은 `closeBy`를 공유하지 않고, 동시 비용은 한 인과적 SCAR로 묶으며 닫는 회차에는 기존 ID의 `paid|transformed`를 우선한다.

#### Terminal exception: vol12/11-epilogue.md

- open WAGER를 resolve하고 지속 SCAR를 확정한다. 종결만을 위해 새 irreversible Gamble을 만들 필요는 없다.
- COVENANT 이후 세계·관계·책임의 변화를 보여 주는 **새 Afterimage 정보**와 현우·아린·ZERO·핵심 ALLY RELAY의 감정적 종결이 필요하다.
- 다음 회차가 없으므로 next-episode Hook은 두지 않는다.
- 상태에는 **episode RELAY: terminal**, manifest에는 **relayTo: null**을 기록한다.
- terminal SCAR는 open으로 남기지 않고 paid 또는 transformed로 닫는다. 비용이 사라졌다는 뜻은 아니다.

### 9.6 악당 지능과 증거

- 서혜진은 3권 감지 뒤 현우를 인간 반례 실험으로 의도적으로 남기고, 화면 밖에서도 관찰·대비를 갱신한다.
- 9권 root 임계 초과 뒤에는 관찰을 고집하지 않고 즉시 회수 단계로 전환한다.
- 악당은 확보하지 않은 정보를 작가 편의로 알지 않는다. 한 번 놀랐으면 다음 비트에는 학습한다.
- 체포된 악당의 자백, 장광설, 갑작스러운 양심으로 진실과 승리를 동시에 해결하지 않는다.
- 시스템 로그·코드·상태표는 회차의 8% 이하이며, 동일한 정보를 산문으로 재설명하지 않는다.
- 같은 문장 첫머리·감탄·침묵·화면 묘사를 관성적으로 반복하지 않는다. 의도적 반복은 의미·주체·결과를 바꾼다.

## 12. 상태 추적 규칙

원고보다 먼저 `state/episode-manifest.json`의 PRD 고정 스키마를 확정하고, 원고 작성 전후로 여섯 state 문서를 함께 갱신한다. 아래는 새 최상위 필드가 아니라 manifest 필드를 각 원장에 투영하는 규칙이다.

```text
[CAPABILITY TREE / volNN chNN]
zeroMode: off | manual | tier1-read | tier2-reversible | tier3-counterfactual | tier4-orchestrate | root
stateDelta.capability: Tier, data scope, action permission, confidence, human approval, 실행 결과
failureType: { class, manifestation }
```

```text
[POWER COST LEDGER / volNN chNN]
TRACE: { Trace, Resource, Agency, Connection, Externality, status }
SCAR: { id, change, status, closeBy } # open은 1~2회 뒤의 서로 겹치지 않는 정경 ID, paid/transformed는 null
```

```text
[ROMANCE GRAPH / volNN chNN]
stateDelta.relationship: 관계 단계, 직접 확인한 동의와 경계, 공개·회수된 접근권
allyRelay: [{ node, stage, choice, cost }]
```

```text
[WAGER / SCAR / RELAY / volNN chNN]
WAGER: { id, mode, stake }
SCAR: { id, change, status, closeBy } # open은 1~2회 뒤의 서로 겹치지 않는 정경 ID, paid/transformed는 null
relay: 다음 회차 Want·Adversary를 발생시키는 결과
relayTo: 정확한 다음 episode ID | V12E11에서만 null
seeds: [{ id, action, deadline }]
allyRelay: [{ node, stage, choice, cost }]
```

## 13. 금지사항

- 실제 해킹·금융범죄·사회공학의 실행 가능한 구체적 절차 서술 금지.
- 여주·조연 외모·성별·직업의 조롱·대상화 금지.
- 제로를 만능 데우스 엑스 마키나로 사용 금지. 제로도 틀리고, 모르고, 대가를 치른다.
- 갈등 없이 능력만 자랑하는 장면 금지.
- 경쟁 유저·악당을 순수 악으로만 소비 금지. 각자의 논리를 준다.
- 개발/AI 밈이 감정선을 압도하는 것 금지.
- 능력이 편의상 갑자기 세지는 파워 인플레 금지. 언락에는 계기가 있다.
- 현우를 선택받은 자로 만들거나 민재호를 ZERO 단독 창조자로 쓰는 것 금지.
- Opacity Key를 투명화·은신·삭제 능력으로 쓰거나 ZERO가 아린 개인 모델을 우회 생성하는 것 금지.
- 공개 페르소나·가명·CASE 라벨로 Opacity Key를 우회하는 데 성공시키는 것 금지. 5권 CASE A에서 ZERO는 동일 대상을 재식별하자 개인 모델 생성 없이 즉시 거부한다.
- Tier 3을 독심·감정 판독·개인 연애 예측으로 쓰는 것 금지.
- 윤가람을 기자로 바꾸는 것 금지. 외부 포렌식 데이터 분석가로 유지한다.
- 아키텍트를 무능하게 방치시키거나 자백 하나로 무너뜨리는 것 금지.
- 6권 이후 삼각관계·질투용 제3자·반복 이별로 로맨스를 지연하는 것 금지.
- 결말에서 현우에게 root를 남기거나 ZERO의 광역 권한을 복구하는 것 금지.
- 서혜진을 죽음·참회·가족 화해로 면책하는 것 금지. 공개 증언과 재판, 용서 비보장을 유지한다.
