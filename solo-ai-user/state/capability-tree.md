# 능력 트리 (Capability Tree)

[시리즈 홈](../README.md) | [문서 목차](../README.md#핵심-문서) | [다음 상태 문서 →](./power-cost-ledger.md)

> 목적: ZERO의 능력을 단일 파워 레벨이 아니라 권한·데이터·확신·인간 승인까지 포함한 계약으로 고정한다. 이 문서에 없는 능력과 우회는 원고에 추가하지 않는다.

## 유효 능력 공식

ZERO의 유효 능력은 다음 다섯 축의 곱이다.

**Tier × data scope × action permission × confidence × human approval**

한 축이라도 0이면 실행도 0이다. 높은 confidence는 데이터 적법성·행동 권한·인간 승인을 대신하지 못한다.

| 축 | 질문 | 허용 상태 | 0이 되는 경우 |
| --- | --- | --- | --- |
| Tier | 어떤 종류의 작업인가 | Tier 1~4 | 정의되지 않은 기능 또는 회수된 기능 |
| data scope | 어떤 데이터를 볼 수 있는가 | 제공 자료, 로컬 연결, 동의된 풀, 허가된 광역 범위 | 자료 부재, 출처 불명, 동의 철회 |
| action permission | 어디까지 행동할 수 있는가 | 관찰, 제안, 제한 실행, 명시된 다중 도구 조정 | 권한 없음, 범위·기간 만료 |
| confidence | 근거가 얼마나 충분한가 | 0.00~1.00과 근거 출처 | 검증 불가, 상충 근거 해소 실패 |
| human approval | 누가 책임 있게 승인했는가 | 회차 승인, 위임 승인, 직접 당사자 동의, 복수 수탁자 승인 | 승인 없음, 당사자 거부, 인간 veto |

## Tier 정의

| 티어 | 이름 | 가능한 것 | 상시 한계 |
| --- | --- | --- | --- |
| Tier 1 | Assist/Collaborate | 허가 자료 분석·요약·초안·가설 제시 | 판단·발표·책임은 인간 몫이며 오염 데이터에서 환각한다 |
| Tier 2 | Automate | 반복 작업·모니터링·파이프라인 대행 | 범위·기간·중지 조건이 필요하고 자동화 흔적이 남는다 |
| Tier 3 | Simulate | 시장·교통·협상 조건·사건 순서의 시스템 반사실 비교 | 개인 마음·연애 결과·숨은 의도 모델링은 금지한다 |
| Tier 4 | Orchestrate | 권한이 다른 복수 도구·에이전트·시스템의 제한 조정 | 각 주체의 거부권을 덮지 못하며 TRACE와 실패 위험이 최대다 |

### root 권한 상태

root는 Tier 5가 아니다. 데이터·도구·행동 범위를 크게 여는 별도 권한 상태이며 같은 다섯 축과 인간 승인 제약을 받는다.

- 9권에서 현우의 우회가 PALISADE root 임계치를 넘어 외부효과와 정당성 위기를 만든다.
- 11권에서 서혜진의 authority recall이 현우의 통제 가정을 무너뜨린다.
- 12권에서 현우의 단독 root는 복구용 사본 없이 재결합 불가능한 12개 승인 권한으로 분해된다.
- COVENANT 창설은 12개 노드 전원 동의가 필요하다. 창설 뒤 고위험 행동은 8/12 승인, 직접 당사자의 행동별 동의, 즉시 철회, 범위·만료 영수증을 모두 요구한다.
- 지정 인간 검토자 한 명의 veto는 행동을 중단할 수 있지만 단독 허가권은 아니다.

## Opacity Key 불변식

- Opacity Key는 서아린을 보이지 않게 하거나 원자료를 삭제하지 않는다.
- 사람과 카메라는 아린을 관찰할 수 있고, 아린이 제공한 자료로 공동 작업할 수 있다.
- ZERO는 아린의 감정·의도·미래 행동에 대한 개인 모델링을 시작하지 않는다.
- CASE A처럼 공개 페르소나를 새 이름으로 부르는 우회도 동일 인물로 재식별하는 즉시 거부한다.
- 거부는 성능 실패가 아니라 human approval 축을 0으로 만드는 준법 행동이다.

## 권별 능력과 고정 실패 회전

| 권 | 능력 국면 | 고정 실패 종 | 인간이 맡는 판단 |
| ---: | --- | --- | --- |
| 1 | Tier 1 협업과 단발 자동화 맛보기 | 오염 데이터 | 오염 입력을 의심하고 근거표와 현장을 다시 확인한다 |
| 2 | Tier 2 자동화와 첫 외부 운영 | 운영·Goodhart | 과대 수요 예측의 재고·노동 비용을 당사자와 함께 감당한다 |
| 3 | Tier 2 광역 관측 확장 | 적대적 정보오염 | 서로 독립된 현장 2인 확인과 원본 인계로 오염을 가른다 |
| 4 | Tier 3 시스템 반사실 진입 | 접근권·블랙스완 | 합법 접근권 밖의 원본과 당사자 결정을 보존한다 |
| 5 | Tier 3의 사회 시스템 적용 | 사회적 수행성 | CASE A 침해를 공개하고 아린의 접근권 회수와 집단 철회를 수용한다 |
| 6 | 기존 기능의 자원·컨텍스트 붕괴 | 자원 붕괴 | 음성·종이·몸의 수동망으로 운영과 관계를 다시 잇는다 |
| 7 | Tier 4 조짐과 라이벌 충돌 | 상대 적응 | 더 빠른 수보다 피해자의 비공개·대피 선택을 우선한다 |
| 8 | Tier 4 다중 조정 | 다중 에이전트 충돌 | 아린과 조연들이 역할 경계를 정하고 인간 구조 RELAY를 만든다 |
| 9 | root 임계 우회 | 외부효과·정당성 | 비참여자 비용을 인정하고 범위를 줄이며 피해자 조건을 수용한다 |
| 10 | ZERO 기원과 자기참조 | 자기참조·정체성 | 선택받음 서사를 거부하고 사람·도구의 독립 거부를 받아들인다 |
| 11 | 최대 출력과 권한 회수 | 권한 회수 | 공개 책임, 물리적 증거 전달, 대안 제도로 패배를 전환한다 |
| 12 | 단독 root 포기와 분산 | 정답 없는 규범 문제 | 복수 시민의 취소 가능한 합의로 하나의 정답을 대체한다 |

## 권별 상태 전진

| 권 | 시작 상태 | 종료 상태 |
| ---: | --- | --- |
| 1 | 모든 축의 존재를 모른다 | Tier 1과 인간 최종 판단의 차이를 배운다 |
| 2 | 회사 안의 제한 협업 | Tier 2를 밖에 쓰고 TRACE와 운영 책임을 남긴다 |
| 3 | 온라인 데이터 신뢰 | 오염을 인정하고 현장·아카이브 교차확인을 능력 계약에 넣는다 |
| 4 | 예측 성공을 권한으로 착각 | 접근권과 Opacity 거부가 성능보다 우선함을 받아들인다 |
| 5 | 공개 데이터면 모델링 가능하다고 오판 | CASE A 재식별 즉시 거부와 당사자 접근권 회수를 겪는다 |
| 6 | 여러 과업을 동시에 유지 | 자원 붕괴 뒤 수동 운영과 명시적 관계 동의를 배운다 |
| 7 | 자신만의 치트라는 환상 | 적응하는 상대와 다른 사용자의 거부권을 인정한다 |
| 8 | 중앙 조정이 최선이라는 믿음 | 역할 경계와 독립 ALLY RELAY 없이는 Tier 4가 무너짐을 확인한다 |
| 9 | 좋은 목적의 우회 정당화 | root 행동의 외부효과·정당성 비용과 회수 전환을 일으킨다 |
| 10 | 기원이 권한을 정당화할 것이라는 기대 | 시민 수탁자 팀의 공동 fork와 우연한 분실을 확인한다 |
| 11 | 최대 출력이면 통제를 되찾을 수 있다는 기대 | authority recall로 패배하고 COVENANT 필요성을 받아들인다 |
| 12 | 단독 root를 되찾을 유혹 | root·광역 권한·부팅 전 기원 기억을 잃고 Tier 1 협업만 유지한다 |

## 회차 기록 규칙

episode manifest의 능력 상태는 최상위 `zeroMode`와 `stateDelta.capability`로만 기록한다. `zeroMode`는 `off|manual|tier1-read|tier2-reversible|tier3-counterfactual|tier4-orchestrate|root` 중 하나이며, `stateDelta.capability`는 `능력:`으로 시작해 Tier·data scope·action permission·confidence·human approval·실행 결과의 세부 계약을 명시한다. 같은 뜻의 임의 최상위 `tier`, `permission`, `result` 별칭을 만들지 않으며, 권한이 없는데 성공한 장면을 결과론으로 정당화하지 않는다.

능력 실패는 비어 있지 않은 `failureType: { class, manifestation }`로 장면 발현까지 적는다. `class`는 위 회전표의 `오염 데이터|운영·Goodhart|적대적 정보오염|접근권·블랙스완|사회적 수행성|자원 붕괴|상대 적응|다중 에이전트 충돌|외부효과·정당성|자기참조·정체성|권한 회수|정답 없는 규범 문제`만 쓴다. 큰 능력 사용은 [power-cost-ledger.md](./power-cost-ledger.md)의 `TRACE`와 연결하고, 조연이 자기 판단으로 권한을 잇거나 거부한 장면은 `{ node, stage, choice, cost }` 구조의 `allyRelay`로 [ally-payoff-ledger.md](./ally-payoff-ledger.md)에 연결한다.
