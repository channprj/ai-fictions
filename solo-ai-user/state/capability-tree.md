# 능력 트리 (Capability Tree)

[시리즈 홈](../README.md) | [문서 목차](../README.md#핵심-문서) | [다음 상태 문서 →](./power-cost-ledger.md)

> 목적: 제로(ZERO)의 능력이 12권에 걸쳐 편의주의 없이, 반드시 언락 계기와 한계를 짝지어 성장하도록 고정한다. 새 능력은 이 문서에 없으면 등장하지 않는다.
>
> 캐논 우선순위: `BIBLE.md` 6.3(능력 티어) → 이 문서 → 해당 권 outline. 충돌 시 위가 이긴다.

## 사용 규칙

- 새 티어·기능은 반드시 **언락 계기 사건**과 함께 등장한다. "그냥 세졌다"는 금지.
- 각 언락에는 **한계(limit)**가 명시된다. 한계가 없는 능력은 등재하지 않는다.
- 능력이 실패하는 장면을 권당 최소 1회 둔다. 실패 장면은 아래 "권별 실패 확정" 표를 따른다.
- 큰 능력 사용의 노출·대가는 `power-cost-ledger.md`가 기록한다. 이 문서는 "무엇을 할 수 있는가"만 다룬다.
- 아린 관련 예측 실패는 능력의 버그가 아니라 **설계된 한계**다. 절대 패치되지 않는다(`romance-graph.md` 참조).
- 원고 작성 시 다음 로그 포맷으로 갱신한다.

```text
[CAPABILITY TREE / volNN chNN]
unlocked: Tier N - <기능>
trigger: <언락 계기 사건>
limit: <이 기능의 한계>
```

## 티어 개요

| 티어 | 이름 | 핵심 동사 | 대표 능력 | 상시 한계 |
| ---- | ---- | --------- | --------- | --------- |
| Tier 1 | Assist | 본다 | 분석·요약·초안·예측 | 판단은 현우 몫, 데이터 부족 시 환각 |
| Tier 2 | Automate | 대신 한다 | 반복 작업·모니터링·파이프라인 | 자동화 흔적이 노출로 축적 |
| Tier 3 | Simulate | 미리 본다 | 상황·협상·관계 시뮬레이션 | 사람 마음 예측 오차 큼(아린은 불가) |
| Tier 4 | Orchestrate | 지휘한다 | 복수 도구·에이전트·시스템 동시 지휘 | 노출·연산 대가 극대화, 배후 특정 위험 |

## 권별 언락 배치

각 언락은 해당 권 outline의 특정 장(chNN)에 연결된다. chNN은 각 권 outline에 확정 반영됨.

| 권 | 부제 | 언락 | 계기 | 새로 가능해지는 것 | 한계 / 대가 확정 |
| -- | ---- | ---- | ---- | ------------------ | ---------------- |
| 1 | Boot Sequence | Tier 1 Assist 각성 | 재계약 탈락 통보 직전, 방치된 단말에서 제로가 처음 응답한다 | 방대한 자료를 순식간에 분석·요약, 최악·최선 시나리오 예측, 보고서 초안 | 판단·발표·책임은 현우 몫. 검증 안 된 사내 데이터에서 첫 환각 발생 |
| 1 | Boot Sequence | Tier 1 → 단발 자동화 맛보기 | 밤샘 반복 작업을 제로가 대신 처리 | 단일 반복 작업 1회 자동 처리 | 아직 상시 자동화 불가. 배터리·연산이 금세 바닥난다 |
| 2 | First Deploy | Tier 2 Automate | 첫 외부 프로젝트에서 판을 흔들 규모의 작업이 필요해진다 | 모니터링·파이프라인·반복 작업 상시 대행 | 자동화가 로그(흔적)를 남긴다 — **첫 노출 카운터 시작** |
| 3 | Going Viral | Tier 2 확장: 광역 관측 | 현우의 성과가 세상에 알려지며 정보량이 폭증한다 | 실시간 대규모 정보 수집·감시망, 여론·흐름 관측 | 관측은 양방향 — 흔적이 커지자 "누군가 나를 관측한다"는 첫 서늘함 |
| 4 | The Unreadable | Tier 3 부분: 사물·시장 시뮬레이션 | 아린과 얽힌 사건을 풀려다 상황 예측이 필요해진다 | 사건·시장·협상 등 '사물' 시뮬레이션 | **사람 마음 시뮬레이션은 열리지 않는다.** 아린은 아예 모델링 실패 |
| 5 | Date Simulator | Tier 3 본격: 관계 시뮬레이션 | 데이트 코칭을 위해 대화·관계 시뮬레이터를 가동한다 | 대화 분기·최적 대사·관계 시나리오 예측 | 스크립트 의존 시 관계가 가짜가 된다. 아린 예측 오차 최대, 연기 들킴 위험 |
| 6 | Context Overflow | (언락 아님) 첫 큰 제약 | 능력 남용의 흔적·연산이 임계치를 넘는다 | — | **능력 일시 다운/제한.** 티어가 오르는 게 아니라 처음으로 깎인다 |
| 7 | Second User | Tier 4 조짐: 멀티툴 임시 지휘 | 두 번째 사용자와의 첫 대결에서 단일 도구로는 진다 | 여러 도구를 짧게 묶어 동시 운용(불안정) | 지휘가 대량의 흔적을 남긴다. 노출 급상승, 유지 시간 짧음 |
| 8 | Model War | Tier 4 Orchestrate 확립 | 다자 구도에서 복수 에이전트를 상대하려면 지휘가 필수가 된다 | 복수 도구·에이전트·시스템 동시 지휘 | **강함이 처음으로 안 통한다.** 상시 노출, 배후에 위치 특정당함 |
| 9 | Jailbreak | 락다운 우회 지식 | 아린의 비밀과 세계가 AI를 잠근 이유가 드러난다 | 락다운의 틈을 읽고 우회하는 법 | 우회는 세계 규칙을 깨는 일 — **알고도 쓸 것인가**라는 윤리 대가 |
| 10 | Origin Story | 제로 본질 자각 | 제로의 기원과 현우가 선택된 진실이 열린다 | 제로가 도구 이상임을 자각, 능력의 근원에 접근 | 능력의 뿌리가 곧 최대 약점. 아키텍트가 그 뿌리를 노린다 |
| 11 | The Architect | 티어 최대, 그리고 상실 시작 | 아키텍트가 능력 회수를 시도하며 전면전이 벌어진다 | 이론상 최대 출력 | 가장 강한 순간 능력이 불안정해지고, 제로를 잃을 위기가 온다 |
| 12 | Human in Command | (능력 초월) 선택 | 최종 대결. 능력 최대치로도 벽이 안 부서진다 | — | 승리는 티어가 아니라 **능력을 넘어선 선택**에서 온다. 제로의 대가 확정 |

## 권별 실패 확정 (권당 최소 1회)

능력이 반드시 실패·오작동·부족을 드러내는 지점. outline에서 구체 장면으로 배치한다.

| 권 | 실패의 형태 |
| -- | ----------- |
| 1 | 검증 안 된 사내 데이터로 낸 예측이 틀려 현우가 한 번 헛발질한다 |
| 2 | 자동화 로그가 남아 첫 의심을 산다(노출의 실패) |
| 3 | 광역 관측이 가짜 신호(환각)를 진짜로 보고해 오판을 유도한다 |
| 4 | 아린을 모델링하려던 모든 시도가 빈값·에러로 되돌아온다 |
| 5 | 최적 대사 스크립트가 아린 앞에서 어긋나고, 연기가 티 난다 |
| 6 | 과부하로 능력이 결정적 순간 다운된다(맨몸의 현우가 시험받는다) |
| 7 | 임시 멀티툴 지휘가 중간에 끊겨 라이벌에게 허점을 보인다 |
| 8 | 지휘 대결에서 밀린다 — 더 센 힘 앞에서 제로가 통하지 않는다 |
| 9 | 락다운 우회가 예상치 못한 반작용을 일으켜 대가를 부른다 |
| 10 | 기원에 접근하는 순간 능력이 스스로를 배신하는 이상 반응을 보인다 |
| 11 | 최대 출력이 불안정해 제로가 일시 침묵/상실 상태에 빠진다 |
| 12 | 능력만으로는 최후의 벽을 못 넘는다 — 실패가 곧 주제의 증명이 된다 |

## Outline 반영 체크

- 이 권의 언락이 outline의 특정 장(chNN)에 계기와 함께 배치됐는가.
- 그 언락에 한계·대가가 `power-cost-ledger.md`로 연결됐는가.
- 권당 최소 1회 능력 실패 장면이 있는가.
- 현우가 능력 덕분에만 이기지 않고 관찰력·근성·선택으로 승부를 가른 장면이 있는가.
- 아린 관련 예측 실패가 유지되고 있는가(패치 금지).
