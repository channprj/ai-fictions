# 제 72화: 재발 방지 패치

[← 이전화: 제 71화](../vol03/ep071.md) | [목차](./README.md) | [다음화: 제 73화 →](../vol03/ep073.md)


사후 분석이 끝나면, 그 다음은 정해져 있다.

재발 방지 패치.

장애를 한 번 막은 운영자는 영웅 대접을 받는다. 같은 장애를 두 번 막은 운영자는 욕을 먹는다. 왜 안 고쳤냐고. 그래서 좋은 운영자는 불을 끈 다음에 불씨가 못 살아남는 환경을 만든다. 인시던트 리포트 마지막 줄에 박히는 그 항목. Action Item — 재발 방지.

어젯밤 엘리스·마리·세리아 앞에 타임라인을 펼쳐놓고 다섯 사건의 지문이 전부 한 손에서 나왔다는 걸 공유했다. 영지 봉인, 그리드 정찰, 아카데미 던전 디도스, 케르베로스 무적 패치, 영지 웜. 우연이 다섯 번이면 그건 설계다. 그 손이 누구인지까진 다 말 안 했지만, 결론 하나는 박았다.

저쪽은 또 온다. 그리고 다음에 올 땐 내가 이 영지에 없을 가능성이 높다. 코어는 수도에 있고, 나는 곧 수도로 가야 하니까.

```
arsian@lumen-territory:~$ cat /var/incident/postmortem-073.md | tail -5

## Action Items
[ ] AI-01  영지 방어 인프라 영속화 (운영자 부재 대응)
[ ] AI-02  C2 재출현 자동 탐지 (점멸 신호 모니터링)
[ ] AI-03  자가 복구 / 자동 점검 (cron)
[ ] AI-04  운영 책임자 지정 (on-call)
[ ] AI-05  생활 인프라 자동화 (영지민 정주율)
```

다섯 줄. 오늘 할 일이다.

핵심은 단순하다. 지금 이 영지는 내가 콘솔 붙잡고 있어야 굴러간다. 오토스케일러도, 터렛 정책도, 골렘 워커 제어권도 전부 내 root 세션에 묶여 있다. 내가 빠지면 이 영지는 운영자 잃은 서버가 된다. 한동안은 관성으로 돌겠지만, 첫 장애가 나는 순간 받아줄 사람이 없다.

이건 좋은 시스템이 아니다. 좋은 시스템은 운영자가 죽어도 돌아간다. 사람한테 의존하는 인프라는 인프라가 아니라 인질극이다. 전생에 제일 싫어했던 게, 자기만 아는 레거시를 끌어안고 "나 없으면 이 서버 못 돌려요" 하며 휴가도 못 가던 그 선임이었다. 그 인간이 맹장 터져 입원한 주에 서버가 어떻게 됐는지는, 아직도 위가 쓰리다.

나는 그런 운영자가 안 된다. 빠지되, 빠져도 영지가 안 죽게 만들고 빠진다.

손목 콘솔을 펼쳤다.

"오늘부로 이 영지를, 사람 손 없이 굴러가는 서버로 만든다."

---

먼저 AI-03. 자동 점검.

운영자가 부재중일 때 시스템이 죽는 가장 흔한 이유는 거창한 공격이 아니다. 사소한 게 쌓여서 곪는 거다. 마나 회로에 먼지 끼고, 터렛 마법진 하나가 슬그머니 멈추고, 채굴 골렘 한 기가 협곡에 끼어서 무한 대기 걸리고. 하나하나는 별거 아닌데, 아무도 안 보면 일주일 뒤에 영지 절반이 멈춘다.

이걸 사람이 매일 순찰 돌면서 막는다? 그게 바로 인질극이다. 사람은 까먹고, 사람은 자고, 사람은 휴가를 간다. 그래서 점검은 사람이 아니라 시계가 한다.

`cron`.

```
arsian@lumen-territory:~$ crontab -e
```

```cron
# /etc/crontab — 루멘 영지 정기 점검. 사람 안 깨우는 야간 운영팀
0 * * * *    healthcheck.sh --all        # 매시: 전 인프라 헬스체크
*/10 * * * * ping.sh --target north       # 10분마다: 북쪽 경계 핑
0 4 * * *    gc.sh --mana --sweep         # 새벽4시: 마나 회로 청소
0 5 * * *    golem-rollcall.sh --unstick  # 새벽5시: 골렘 점호/끼임 해소
0 3 * * 0    snapshot.sh --full           # 일요일: 전체 스냅샷 백업
```

새벽 4시에 마나 회로 청소, 새벽 5시에 골렘 점호. 사람이 자는 시간에 시계가 일한다. 매시 정각 전 인프라가 자기 손목을 짚고, 10분마다 북쪽 경계가 살아 있다고 핑을 친다.

영지민들은 모를 거다. 자기들이 자는 새벽에 보이지 않는 손이 영지를 한 바퀴 돌며 먼지를 쓸고 끼인 골렘을 빼준다는 걸. 그게 좋은 인프라다. 운영자가 일 잘하면 운영자가 한 일이 안 보인다. 그게 칭찬이다.

엘리스가 옆에서 그 스케줄 표를 노려봤다.

"이거… 정해진 시각마다 자동으로 실행되는 거지? 영창도 없이, 시전자도 없이?"

"시전자는 시계야. 시각이 곧 트리거고."

"술식이 사람을 안 거치고 시간만 보고 발동한다고? 그런 마법은 교본에 없어."

"교본이 안 늦은 게 아니라 8년쯤 늦은 거지."

엘리스가 또 양피지에 받아 적었다. '시각 = 트리거. 시전자 불필요. 교본 부재.' 이 인간 QA 노트는 이제 거의 내 운영 매뉴얼 초판이었다.

AI-03의 후반부, 자가 복구.

점검은 문제를 발견하는 거고, 복구는 발견한 문제를 사람 없이 고치는 거다. 둘은 다르다. 새벽 4시에 골렘 한 기가 멈춘 걸 발견만 하고 아침까지 그냥 둔다? 그럼 그건 점검이 아니라 그냥 알람 시계다. 발견하면 그 자리에서 고쳐야 한다.

전생 용어로는 셀프 힐링. 자가 치유.

```
arsian@lumen-territory:~$ vi /opt/lumen/healthcheck.sh
```

```bash
# healthcheck.sh — 발견하면 바로 고친다. 사람 안 부른다.
for node in $(lumen-nodes --all); do
  case "$(probe "$node")" in
    HEALTHY)      continue ;;                    # 정상이면 통과
    UNRESPONSIVE) kill -9 "$node"; respawn "$node" ;;  # 죽이고 재기동
    STUCK)        unstick "$node" ;;             # 끼였으면 빼준다
    DRIFTED)      apply --config /etc/lumen/golden ;; # 원본으로 복원
    *)            page "@on-call: $node" ;;       # 모르는 상태만 사람 호출
  esac
done
```

원칙은 하나다. 알려진 고장은 기계가 알아서 고치고, 모르는 고장만 사람을 깨운다.

응답 없는 노드는 `kill -9` 후 재기동. 죽었다 다시 켜면 멀쩡해지는 게 IT의 90퍼센트다. 끼인 노드는 빼주고, 설정이 틀어진 노드는 골든 이미지로 강제 복원. 누가 손대서 망가뜨려도 새벽이면 원본으로 돌아와 있다.

마지막 `*)` 케이스만 사람을 깨운다. 케이스로 안 잡히는 미지의 장애. 그것만 호출이 나간다. 사람을 깨우는 알람은 비싸야 한다. 매일 울리는 알람은 소음이고, 소음은 사람이 무시하는 순간 진짜 장애를 놓치게 만든다.

마리가 어깨너머로 새카만 글자들을 들여다보며 고개를 갸웃했다.

"도련님, 이 검은 주문이… 영지를 자기가 알아서 고친다고요? 사람이 안 고쳐도요?"

"어. 골렘 하나 멈추면 시스템이 알아서 발로 차서 다시 돌려. 너 안 깨우고."

"그럼 도련님 안 계셔도… 영지가 안 멈춰요?"

날카로운 질문이었다. 호들갑쟁이 메이드가 핵심을 찔렀다. 오늘 작업의 목적을 한 줄로 요약한 셈이다.

"그게 오늘 만드는 거야. 내가 없어도 안 멈추는 영지."

마리의 표정이 살짝 어두워졌다. '내가 없어도'라는 말의 뉘앙스를 알아챈 거다. 이 메이드는 호들갑은 떨어도 눈치는 빠르다.

AI-02. C2 재출현 탐지.

이게 어젯밤 사후 분석에서 나온 가장 중요한 항목이다. 다섯 사건 전부 같은 손이 친 거라면, 그 손은 또 온다. 그리고 그 손이 다시 올 때 제일 먼저 켜질 건, 북쪽 능선에서 또박또박 점멸하던 그 C2 신호일 거다. 이미 한 번 봤으니까, 두 번째는 못 알아볼 수가 없다.

지난번엔 세리아가 "신호를 주고받는 것 같다"고 육안으로 알아챘다. 운 좋게도. 하지만 운에 의존하는 탐지는 탐지가 아니다. 다음엔 사람이 안 보고 있을 때 켜질 거다. 그래서 사람 눈 대신 센서를 박는다.

```
arsian@lumen-territory:~$ vi /etc/lumen/ids/c2-watch.rules
```

```
# c2-watch.rules — C2 재출현 침입 탐지(IDS). 지난 점멸 패턴을 시그니처로 박음
rule "C2_BEACON_REAPPEAR" {
  match:     periodic_signal(interval=~3s, source=north_ridge)
  signature: gemini_fingerprint        # raw max-int, 게으른 들여쓰기
  on_match:
    1. snapshot  --target $source       # 증거 즉시 보존
    2. iptables  -s $source -j DROP     # 일단 회선부터 끊는다
    3. page      @on-call --priority P0 # 운영 책임자 호출
    4. autoscale --preheat north --x2   # 방어선 선제 가열
}
```

이건 '발견하면 알린다'가 아니다. 발견하면 일단 끊고, 증거를 박제하고, 그 다음에 알린다. 순서가 중요하다.

지난번엔 C2를 역추적하고 분석하고 확신한 다음에야 회선을 끊었다. 그 사이 웜은 100만 단위까지 불었다. 이번엔 안 그런다. 패턴이 잡히면 묻지도 따지지도 않고 회선부터 `DROP`한다. 끊고 나서 분석한다. 멈춰놓고 따지는 게, 따지면서 당하는 것보다 백배 낫다.

그리고 마지막 줄, 방어선 선제 가열. C2가 보이면 웜이 곧 온다는 뜻이니, 적이 한 마리 솟기 전에 오토스케일러를 미리 두 배로 데워둔다. 비 오기 전에 우산을 펴두는 거다.

```
arsian@lumen-territory:~$ ids-ctl reload && ids-ctl status
[INFO] rule loaded: C2_BEACON_REAPPEAR (severity=CRITICAL)
[INFO] sensor armed on: north_ridge
[INFO] watching for: gemini_fingerprint
[OK]   IDS active. waiting silently.
```

센서가 북쪽 능선을 조용히 노려보기 시작했다. 그 점멸이 다시 켜지는 순간, 내가 수도에 있든 자고 있든 술 마시고 있든, 회선은 자동으로 끊기고 방어선은 자동으로 데워지고, 그 다음에야 호출이 나간다.

저쪽이 같은 수법으로 또 오면, 이번엔 시작도 하기 전에 막힌다. 같은 익스플로잇은 한 번만 통한다. 패치된 취약점은 더 이상 취약점이 아니다.

AI-01. 영속화. 그리고 AI-04. 운영 책임자.

이 둘은 한 묶음이다. 자동화가 아무리 잘 돌아도, 그 `*)` 케이스 — 기계가 못 고치는 미지의 장애 — 가 났을 때 콜을 받을 사람이 필요하다. 자동화는 알려진 99퍼센트를 처리하고, 사람은 미지의 1퍼센트를 받는다. 그 1퍼센트가 진짜 비싼 거라, 거기엔 제일 믿을 만한 사람을 박아야 한다.

후보는 처음부터 정해져 있었다.

"세리아."

성벽 아래에서 신입 기사들 훈련을 보던 세리아가 고개를 들었다. 닷새 동안 단일 노드로 웜을 막았던 인간. 1권에서 마나 회로 디버깅해서 거둔 이래로, 한 번도 내 신뢰를 배신한 적 없는 가장 충직한 물리 딜러.

"부르셨습니까, 마스터."

"오늘부터 직함 하나 더 줄게. 영지 수호자 겸 운영 책임자."

세리아의 눈이 살짝 커졌다.

"운영… 책임자요?"

"내가 수도로 가야 해. 한동안 자리를 비울 거야. 그동안 이 영지의 on-call은 너야."

나는 그녀에게 권한 토큰 하나를 건넸다. 정확히는, 그녀의 마나 코어에 제한된 관리자 권한을 새겼다. root는 아니다. root는 나 하나뿐이고, 앞으로도 그럴 거다. 하지만 영지를 지키고 운영하는 데 필요한 만큼의 권한은 줬다.

```
arsian@lumen-territory:~$ useradd serria --role guardian
arsian@lumen-territory:~$ usermod -aG operators,oncall serria
# serria = (defense,golem,turret) NOPASSWD: ALL  ※ root/Creator-level 제외

[SYSTEM] 'serria'에게 영지 운영 권한 부여 (방어/골렘/터렛/영지민)
[SYSTEM] 제외 범위: 월드 커널 / root / Creator-level
[SYSTEM] on-call 등록 완료. P0 호출 수신 대상: serria
```

"네가 할 일은 단순해. 평소엔 자동화가 다 한다. 너는 그냥 훈련 돌리고 영지 살피면 돼. 근데 시스템이 너를 호출하면 — 그건 기계가 못 고치는 일이 났다는 뜻이야. 그땐 네가 가서 칼로 해결해. 칼로 안 되면 나한테 연락하고."

"기계가 못 고치는 일이라면…"

"몰라. 그게 미지수니까 사람을 부르는 거야. 알면 자동화에 박아놨겠지. 미리 다 알 수 있는 장애면 그건 장애가 아니라 일정이고."

세리아가 토큰을 두 손으로 받아 들었다. 그 깐깐하고 충직한 얼굴에 묘한 무게가 실렸다. 검 한 자루로 영지를 지키던 기사가, 영지 전체의 운영을 짊어지는 책임자가 되는 순간이었다.

"마스터. 한 가지만 여쭙겠습니다."

"어."

"제가 이 영지를 지키는 동안, 마스터는 어디서 무얼 하십니까."

나는 잠깐 멈칫했다. 이 인간은 충성스러운 것치고 가끔 정곡을 찌른다.

"불 지른 놈 잡으러. 그놈 본진이 수도에 있거든."

세리아는 더 묻지 않았다. 대신 한쪽 무릎을 꿇고 토큰을 가슴에 댔다.

"이 영지, 마스터께서 돌아오실 때까지 단 한 줄의 에러도 없이 지키겠습니다."

"에러는 날 거야. 안 나는 시스템은 없어. 다만 네가 못 받는 에러는 안 나게 해놨으니까, 너무 비장할 거 없어."

세리아의 비장함에 김을 좀 뺐다. 운영자가 부하한테 '무사고'를 약속받으면 안 된다. 무사고를 약속한 부하는 사고가 났을 때 그걸 숨기니까. 사고는 나도 괜찮고, 다만 빨리 보고하는 문화. 그게 진짜 안정적인 운영팀이다.

마지막. AI-05. 생활 인프라 자동화.

방어만 자동화하고 끝내면 그건 절반짜리다. 영지가 안 무너지는 거랑, 영지에 사람이 살고 싶어 하는 건 다른 문제다. 군대만 있고 살림이 엉망인 영지는, 적이 안 와도 영지민이 알아서 떠난다. 정주율이 떨어지는 서버는 트래픽 없이도 죽는다.

그리고 이 영역의 운영 책임자는, 처음부터 한 명뿐이었다.

"마리."

빨래 바구니를 안고 종종거리던 마리가 화들짝 돌아봤다.

"네, 네! 도련님!"

"너도 직함 줄게. 생활 인프라 책임자."

"새, 생활…? 그게 뭔데요?"

"네가 맨날 하는 거. 빨래, 청소, 부엌, 우물 관리. 그거 이제 너 혼자 손으로 다 하지 마. 자동화 깔아줄 테니까 관리만 해."

나는 영주성 살림 전반에 자동화 마법진을 깔았다. 우물에서 물 길어 올리는 펌프 마법진, 빨래를 정해진 시각에 돌리는 세탁 데몬, 부엌 화덕에 마석 연료를 자동 공급하는 컨베이어. 다 골렘 워커 풀의 유휴 자원을 쪼개서 돌리는 거다. 어차피 새벽엔 채굴이고 방어고 부하가 낮으니까, 그 남는 연산을 살림에 돌린다. 안 쓰는 워커는 그냥 노는 비용이니까.

```
arsian@lumen-territory:~$ kubectl label nodes stone-golem-09{1..9} role=household
[INFO] 9 nodes assigned to: household (off-peak only)
[INFO] cron jobs registered: well_pump, laundry, kitchen_feed

arsian@lumen-territory:~$ systemctl enable --now lumen-household.service
[OK] household automation active.
[INFO] 마리's manual workload: -73%
```

마리의 수작업 부하 마이너스 73퍼센트.

마리가 정해진 시각에 빨래가 저절로 돌아가는 걸 보고 입을 떡 벌렸다.

"도, 도련님! 빨래가… 빨래가 혼자 빨아져요!"

"너 일 줄이라고. 너 하루 종일 뛰어다니다 쓰러질 뻔한 거 한두 번이 아니잖아."

"그, 그럼 저는 이제 뭘 해요?!"

"관리. 펌프 막히면 골렘 부르고, 빨래 안 돌면 나한테 보고하고. 손으로 다 하는 게 아니라, 잘 돌아가는지 보는 거야. 너도 이제 운영자야."

마리가 '운영자'라는 단어를 입속에서 굴려봤다. 그러다 갑자기 눈물이 그렁그렁해졌다. 호들갑쟁이답게.

"도련님… 저 같은 게… 운영자라뇨…"

"같은 게 어딨어. 영지민이 영지에 붙어 있는 건 방어가 튼튼해서가 아니라 살림이 굴러가서야. 우물에서 물 나오고, 부엌에 불 들어오고, 이불이 따뜻하면 사람은 안 떠나. 그거 관리하는 게 영지 정주율을 지키는 거고. 군대보다 어떤 면에선 더 중요해."

마리가 앞치마로 눈을 훔쳤다. 이 메이드의 충성심은 세리아랑 결이 다르다. 세리아가 칼의 충성이면, 마리는 살림의 충성이다. 둘 다 영지를 떠받치는 기둥이라는 건 똑같지만.

---

해가 기울 무렵, 다섯 줄의 Action Item을 전부 닫았다.

```
arsian@lumen-territory:~$ cat /var/incident/postmortem-073.md | tail -7

## Action Items
[x] AI-01  영지 방어 인프라 영속화          ... DONE
[x] AI-02  C2 재출현 자동 탐지 (IDS)         ... DONE
[x] AI-03  자가 복구 / 자동 점검 (cron)      ... DONE
[x] AI-04  운영 책임자 지정 (serria/oncall)  ... DONE
[x] AI-05  생활 인프라 자동화 (mari/household)... DONE

[STATUS] post-mortem CLOSED.
```

체크박스 다섯 개가 전부 닫혔다. 인시던트 리포트를 닫을 때의 그 깔끔한 만족감. 불을 끈 것보다, 불이 다시 안 붙게 만든 게 운영자한텐 더 큰 일이다.

나는 마지막으로 전체 시스템 상태를 한 번 훑었다.

```
arsian@lumen-territory:~$ lumen-status --summary

┌─ LUMEN TERRITORY :: PRIVATE SERVER ────────────────┐
│ uptime          : recovered, ONLINE                │
│ defense         : autoscaler (self-managing)       │
│ monitoring      : cron + IDS (C2 watch armed)      │
│ self-healing    : healthcheck.sh (hourly)          │
│ on-call         : serria (guardian)                │
│ household        : automated (mari)                │
│ operator present : NO  ←←← 운영자 부재 가능        │
│ status           : RUNS WITHOUT OPERATOR  ✅       │
└────────────────────────────────────────────────────┘
```

`RUNS WITHOUT OPERATOR.`

운영자 없이 돌아간다.

이게 오늘 만든 거다. 내가 빠져도 안 죽는 영지. 내가 자도, 내가 술 마셔도, 내가 수도에서 다른 불을 끄고 있어도, 이 영지는 자기가 자기를 운영한다. 점검도, 복구도, 탐지도, 방어도 전부 시계와 규칙과 센서가 한다. 사람은 미지의 1퍼센트만 받는다. 이게 진짜 인프라다. 사람한테 안 묶인 인프라.

깡촌 남작 영지가, 몇 달 만에 진짜 'Private Server'로 완성됐다. 내 손을 떠나도 무너지지 않는, 자기 완결형 시스템.

좋은 자동화는 짜고 나면 운영자가 할 일이 없다.

지금 내가 딱 그 상태였다.

손이 비었다.

---

성벽에 기대 식은 커피를 마셨다. 미지근했다. 늘 그렇듯이.

저 아래로 영지가 보였다. 새벽이면 보이지 않는 손이 마나 회로를 쓸고, 골렘들은 점호를 받고, 우물에선 물이 솟고 빨래가 돌아갈 거다. 북쪽 능선엔 센서가 조용히 그 점멸을 기다리고, 방어선은 적이 안 와도 두 배로 데워질 준비가 돼 있다. 세리아는 칼을 갈며 on-call을 서고, 마리는 살림을 관리한다.

다 자동이다. 내가 없어도 돈다.

운영자의 손이 비면, 하는 일은 정해져 있다. 다음 타깃을 본다.

나는 커피잔을 내려놓고, 고개를 들어 북쪽 너머 — 능선보다 훨씬 먼 곳을 봤다. 안개 너머, 지평선 끝에 희미하게 걸린 거대한 도시의 윤곽. 이 세계의 진짜 코어가 박혀 있는 곳.

수도.

다섯 사건의 모든 회선이 거기로 수렴했다. C2를 역추적하다 끊긴 그 `Creator-level` 라인도, 결국 저 도시 중앙의 한 점으로 모인다. 이 영지를 다섯 번 친 그 손에 닿으려면, 저 도시 중앙의 코어를 거쳐야 한다.

영지는 이제 손을 안 탄다. 방어 인프라는 영속화됐고, 운영팀은 세워졌고, 재발 방지 패치까지 박았다. 여기서 내가 더 할 일은 없다.

손이 빈 운영자는, 더 큰 서버로 출근한다.

"영지 정리는 끝났네."

나는 손목 콘솔을 접으며 중얼거렸다.

"이제 진짜 코어가 어디 박혀 있는지, 가서 직접 까볼 차례지."

수도의 윤곽 위로, 손목 콘솔이 마지막으로 작은 알림 하나를 띄웠다.

```
[INFO] local incident: CLOSED.
[INFO] next target resolved: capital (core suspected)
[TRACE] Creator-level route → converging at: CAPITAL/CENTRAL
[INFO] 다음 작업 대기 중...
```

다음 작업 대기 중.

나는 입꼬리를 올렸다. 영지의 불은 다 껐다. 재발도 막았다. 이제 발신지로 직접 출장 갈 시간이었다.
