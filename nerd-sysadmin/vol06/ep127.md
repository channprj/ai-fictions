# 제 127화: 아이언 포지

[← 이전: 126화: 불청객 사절단](./ep126.md) | [시리즈홈](../README.md) | [목차](./README.md) | [다음: 128화: 프로토콜 mismatch →](./ep128.md)

6권, 오픈소스와 마법 생물 패치. 서드파티 장애의 도입부였다.

그롬을 따라 내려간 아이언 포지는, 도시 전체가 하나의 거대한 공장이었다. 천장까지 닿는 용광로가 정해진 출력을 넘어 시뻘겋게 끓어올랐고, 사람 키 두 배는 되는 골렘들이 벽에 머리를 박고 또 박았다. 같은 동작을 멈추지 못하는 기계는, 무한 루프에 빠진 프로세스와 정확히 같은 얼굴을 하고 있었다.

가장 먼저 할 일은 분명했다. 폭주하는 것부터 멈춰야 했다. 그런데 드워프들이 용광로 앞을 막아섰다. "기계에도 영혼이 있소. 함부로 숨을 끊을 순 없소." 그롬조차 전원을 내리는 것만은 안 된다고 버텼다. 죽어가는데도 끄지 못하게 하는 건 운영자에게 제일 곤란한 부탁이다. 죽이지 않으면서 멈추는 방법을 찾아야 했다.

다행히 멈추는 것과 끄는 것은 다르다. 전원을 뽑아 영혼이고 데이터고 통째로 날려 버리는 대신, 공장 전체를 비상 모드로 한 단계만 내리면 됐다. 위험한 작업만 정지시키고 핵심 기능은 최소 출력으로 살려 두는 것이다. 제국 황제를 삭제하지 않고 잠가만 뒀던 것과 같은 손버릇이었다.

```
root@lumen-control:~# systemctl isolate emergency.target
[SYSTEM] 아르시안은 공장 전체를 안전 모드로 낮춘다.
```

용광로의 굉음이 한순간에 잦아들었다. 머리를 박던 골렘들이 동작 중간에 얼어붙듯 멈췄고, 드워프들은 자기 기계가 '죽지 않았다'는 걸 확인하고서야 숨을 골랐다. 영혼은 멀쩡했다. 다만 위험한 명령만 멎은 것뿐이다. 세리아가 그제야 검을 거뒀고, 마리는 그을음투성이 작업대 위에 식은 커피를 올려 두었다.

그래서 나는 응급 처치가 끝났다는 말 대신, 멈춰 세운 이유부터 기록해 두었다.

```
root@lumen-control:~# printf '%s' '아이언 포지' >> /incident/vol06.log
[SYSTEM] 인시던트 로그가 저장되었습니다.
```

폭주를 멈춘 건 응급 처치였지 치료가 아니었다. 판교에서도 새벽에 서버가 미쳐 날뛰면 일단 트래픽부터 끊어 한숨 돌린 뒤, 날이 밝으면 진짜 원인을 팠다. 멈춰 세운 지금부터가 시작이었다. 멀쩡하던 골렘들이 왜 한꺼번에 같은 미친 짓을 시작했는지, 그 방아쇠를 찾아야 했다.

골렘 하나의 패널에서 API 버전 오류가 찍힌다.

나는 콘솔을 접지 않았다. 안전 모드는 시간을 벌어 줄 뿐, 멈춰 둔 공장을 영원히 세워 둘 수는 없었다. 저 버전 오류 한 줄이 무엇을 가리키는지 읽어내는 순간, 다시 손을 움직여야 했다.
