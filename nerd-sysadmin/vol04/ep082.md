# 제 82화: 부패 귀족 API

[← 이전: 81화: 권한 로그 감사](./ep081.md) | [시리즈홈](../README.md) | [목차](./README.md) | [다음: 83화: 반란 선언 →](./ep083.md)

4권, 왕국 시스템 아키텍처 재설계. 왕자의 백도어의 확장 구간이었다.

권한 로그의 주인을 좁혀 가는 동안, 이번에는 귀족들이 움직였다. 병석의 국왕을 두고 왕자를 지지하라는 요구가 회의장을 돌았다. 명분은 '왕국의 안정'이었지만, 나는 그 말을 들을 때마다 대시보드의 빨간 점을 떠올렸다. 사람이 안정을 입에 올릴 때는 대개 무언가 이미 흔들리고 있을 때다.

엘리스는 회의장을 한 바퀴 돌고 와서 명단을 셋으로 갈랐다. 진심으로 왕자의 편인 자, 바람을 타는 기회주의자, 그리고 그저 겁먹은 자. "셋을 같은 적으로 보면 안 돼요. 다루는 법이 다르니까요." 맞는 말이었다. 하지만 나는 사람보다 그들이 쥔 권한이 더 궁금했다. 귀족의 충성은 변덕스러워도, 그 충성을 사고파는 통로는 어딘가 코드로 박혀 있을 테니까.

왕국의 귀족 특권은 명예가 아니라 시스템 권한이었다. 면세, 징집 면제, 사병 보유, 영지 재판권 — 전부 중앙 코어 어딘가에 호출 가능한 함수로 등록돼 있었다. 누가 왕자에게 붙으면 무엇을 얻는지 알려면, 그 특권이 애초에 어떻게 정의돼 있는지부터 꺼내 봐야 했다.

```
root@lumen-control:~# SELECT house, privilege FROM nobles WHERE bribe=true;
[SYSTEM] 아르시안은 귀족 특권이 하드코딩된 낡은 API 목록을 확보한다.
```

쿼리 한 줄에 수백 년 치 특권이 표로 펼쳐졌다. 어느 가문이 어떤 권한을 언제 받았는지, 그 대가로 무엇을 약속했는지가 그대로 박혀 있었다. 하드코딩. 조건도, 만료도, 회수 절차도 없이 이름 옆에 영구히 붙어 버린 권한들. 이런 건 기능이 아니라 부채다. 엘리스가 표를 보고 숨을 들이켰다. "이게…… 전부 합법이었다고요?" 합법이었다. 그게 더 나빴다.

마리가 식은 잔을 새 커피로 바꿔 놓으며 표를 흘끔 봤다. "도련님, 이걸 다 외우시려고요?" 외울 필요는 없었다. 사람이 외우는 권한은 잊히지만, 코드에 박힌 권한은 잊히지도 사라지지도 않는다. 그게 문제였다.

누가 매수됐는지 따지는 건 나중 일이었다. 먼저 이 목록이 사라지기 전에 박제해 둬야 했다. 특권을 쥔 자들은 추궁이 들어오면 증거부터 지우는 법이니까. 그래서 나는 분노 대신 로그를 저장했다.

```
root@lumen-control:~# printf '%s' '부패 귀족 API' >> /incident/vol04.log
[SYSTEM] 인시던트 로그가 저장되었습니다.
```

로그가 쌓이면 패턴이 보인다. 이 표가 말해 주는 패턴은 분명했다. 왕국은 오랫동안 권한을 회수 불가능한 형태로 발급해 왔고, 그 빚이 한꺼번에 왕자에게 흘러가고 있었다. 귀족들이 부패한 게 아니라, 부패할 수밖에 없도록 설계된 API가 먼저였다. 판교에서 가장 갚기 어려운 빚도 늘 만료도 회수 절차도 없이 내준 권한이었다. 사람은 떠나도 권한은 남고, 남은 권한은 반드시 누군가의 손에 다시 쥐인다. 사람을 갈아 끼운다고 해결될 문제가 아니었다. 함수 자체를 다시 써야 했다.

목록을 갈무리하고 고개를 들었을 때, 회의장 밖이 소란스러웠다. 협박이 통하지 않으리란 걸 안 쪽이 다음 수를 꺼낸 것이다.

나는 콘솔을 접지 않았다. 제1왕자가 왕성 밖에서 군권을 호출한다. 정치로 안 되면 군대로, 백도어가 막히면 정문을 부수겠다는 뜻이었다. 나는 다음 로그가 올라올 자리를 미리 비워 두었다.
