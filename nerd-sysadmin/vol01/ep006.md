# 제 6화: 전부 똥코드잖아

뒷마당이 폐허가 된 다음 날 아침.

나는 아버지와 기사장을 적당히 안심시켜 돌려보내고, 마리한테는 "오늘은 좀 쉬고 싶으니 방해하지 말라"고 일러둔 뒤 방문을 걸어 잠갔다. 핑계였다. 쉬려는 게 아니라, 코드를 까보려는 거였다.

화염 마법 하나가 그 꼴이었다면, 나머지는 안 봐도 뻔했다. 하지만 직접 안 본 코드는 신뢰하지 않는다. 8년차 서버 개발자의 제1원칙이다. "될 거예요"라고 말하는 신입의 PR을 그냥 머지했다가 새벽 3시에 전화받아본 놈은 안다. 검증되지 않은 코드는 전부 잠재적 장애다.

나는 침대에 책상다리로 앉아, 허공에 떠 있는 검은 터미널을 끌어왔다. 커서가 깜빡였다.

```
root@lumen-territory:~# cd /sys/magic/
root@lumen-territory:/sys/magic# ls -al
drwxr-xr-x  fire/
drwxr-xr-x  heal/
drwxr-xr-x  shield/
drwxr-xr-x  buff/
drwxr-xr-x  water/
drwxr-xr-x  light/
-rw-r--r--  README.legacy   (last modified: 312 years ago)
```

"312년 동안 아무도 안 건드린 디렉토리라……."

레거시의 정의 그 자체였다. 312년이면 처음 짠 놈은 무덤에서 백골이 됐을 거고, 그동안 아무도 이 코드를 리뷰하지도, 리팩토링하지도, 심지어 열어보지도 않았다는 뜻이다. 그냥 "돌아가니까 됐다"며 312년을 방치한 것이다.

세상에서 가장 무서운 문장이 뭔지 아는가? "건드리지 마, 그거 잘 돌아가고 있어." 진짜로 잘 돌아가는 코드는 그런 말을 들을 일이 없다. 그 말이 나온다는 건, 아무도 그 안이 어떻게 생겼는지 모른다는 뜻이다.

나는 `heal/` 디렉토리부터 열었다. 가장 기본이고, 가장 많이 쓰이는 마법이니까.

```
root@lumen-territory:/sys/magic# cat heal/cure_light.mag
```

화면을 채운 코드를 보는 순간, 나도 모르게 입에서 욕이 새어 나왔다.

"……시발, 이게 뭐야."

```
function cast_cure_light(caster, target) {
    chant("오 위대한 빛이여", delay: 1500);
    chant("상처받은 자에게 자비를", delay: 1500);
    for (i = 0; i < 9999; i++) {
        mana_buffer.push(allocate_mana(1));   // 마나 버퍼에 계속 쌓기만 함
    }
    apply_heal(target, amount: 30);
    // mana_buffer 해제 코드 없음
    return;
}
```

손이 부들부들 떨렸다. 이건 단순히 느린 코드가 아니었다. 이건 **메모리 릭**이었다.

설명하자면 이렇다. 힐 한 번 쓸 때마다 `mana_buffer`라는 임시 저장 공간에 마나를 9999개씩 쌓아 올린다. 그런데 그걸 다 쓰고 나서 비우는 코드가 없다. 함수가 끝나도 버퍼에 쌓인 마나는 그대로 남는다. 힐을 한 번 쓰면 9999, 두 번 쓰면 19998, 세 번 쓰면 29997…….

새는 거다. 힐을 쓰면 쓸수록 마나가 회수되지 않고 어딘가에 고여서 썩어간다.

"그래서였구나."

나는 머릿속으로 빙의 정보를 더듬었다. 이 세계의 힐러, 사제들. 그들은 하나같이 단명했다. 전장에서 부상병 몇 명만 치료해도 사제가 픽픽 쓰러지고, 나이 마흔만 넘어도 마나 고갈로 폐인이 되는 게 당연시됐다. 사람들은 그걸 "치유의 대가" "성스러운 희생"이라고 불렀다.

희생은 무슨. **메모리 릭으로 시스템 자원이 고갈된 거다.** 사제들은 신성한 대가를 치른 게 아니라, 312년 묵은 똥코드 때문에 마나가 줄줄 새는 채로 평생을 살다 간 거였다.

전생에서 본 적 있다. 서버 한 대가 매일 새벽 4시에 죽어 나가는데, 원인을 못 찾아서 매일 수동으로 재부팅하던 팀. 알고 보니 누가 짜놓은 코드가 메모리를 안 비우고 계속 쌓고 있었던 거였다. 한 줄. 딱 한 줄, 버퍼 비우는 코드 한 줄이 없어서, 그 팀은 1년을 새벽마다 좀비처럼 일어나 서버를 살려냈다.

이 세계의 사제들이 딱 그 꼴이었다. 다만 그들이 재부팅한 건 서버가 아니라 자기 수명이었을 뿐이고.

"고치면 되지."

나는 `vi`를 열었다. 9999짜리 무의미한 루프를 통째로 지웠다. 의미 없는 영창 두 줄도 삭제했다. 그냥 필요한 만큼만 할당하고, 쓰고, 끝나면 반납하게 했다.

```
function cast_cure_light(caster, target) {
    let mana = allocate_mana(2);
    apply_heal(target, amount: 30);
    release(mana);          // 다 썼으면 반납. 당연한 거 아님?
}
```

마나 할당량은 만 단위에서 2로 줄었다. 영창 시간 3초는 0이 됐다. 그리고 가장 중요한 건, 더 이상 마나가 새지 않는다는 것. 이제 이 세계의 사제들은 힐 좀 썼다고 요절하지 않아도 된다.

"한 줄. 진짜 `release` 한 줄이 없어서 312년 동안 사람이 죽어 나간 거라고?"

나는 헛웃음을 터뜨렸다. 화가 나기도 하고, 어이가 없기도 했다. 이 세계를 짠 그 무능한 창조신 새끼는, 코드 리뷰라는 걸 받아본 적이 없는 게 분명했다. 사수도 없이 혼자 우주를 배포한 주니어 개발자. 그게 이 세계의 창조신이었다.

다음. 나는 `shield/` 디렉토리를 열었다.

```
root@lumen-territory:/sys/magic# cat shield/barrier.mag
```

```
function cast_barrier(caster) {
    summon_spirit("방패의 정령");
    sleep(2000);                    // ??? 왜?
    summon_spirit("수호의 정령");
    sleep(2000);                    // ??????
    chant("나를 지키소서", delay: 1000);
    deploy_barrier(strength: 50);
}
```

"또 너냐, `sleep`."

나는 관자놀이를 짚었다. 보호막 마법이다. 적의 공격을 막는, 한순간이 생사를 가르는 방어 주문. 그런데 이 코드는 보호막을 펼치기 전에 무려 5초를 **그냥 잔다.**

`sleep(2000)`. 2초 동안 아무것도 안 하고 멈춰 있는 코드. 그게 두 번. 거기에 영창 1초까지. 총 5초.

전장에서 5초가 어떤 시간인지 아는가? 화살이 날아오는 데 1초, 칼이 목을 베는 데 0.5초다. 이 보호막을 펼치려고 "방패의 정령" 부르고 2초 멍때리고, "수호의 정령" 부르고 또 2초 멍때리는 동안, 시전자는 이미 고슴도치가 됐을 거다.

대체 왜 `sleep`을 넣었을까. 추측건대, 처음 짠 놈이 정령 소환 모션이 끝날 때까지 기다리려고 임시로 박아둔 거다. "일단 2초 기다리면 되겠지" 하고. 그러고는 그대로 312년. 임시방편이 영구방편이 되는 클래식한 패턴이다.

전생에서도 봤다. 비동기 처리하기 귀찮으니까 `Thread.sleep(2000)` 박아놓고 "나중에 고칠게요" 하던 동료. 그 "나중에"는 영원히 오지 않았고, 결국 그 코드는 운영에 올라가 사용자 만 명을 2초씩 기다리게 만들었다.

```
function cast_barrier(caster) {
    let mana = allocate_mana(3);
    deploy_barrier(strength: 50);   // 즉시 전개. 자는 시간 없음.
    release(mana);
}
```

`sleep`을 다 걷어내고, 의미 없는 정령 소환 의식도 지웠다. 보호막은 이제 0초 만에 펼쳐진다. 무영창 즉시 방어. 화살이 날아오면 날아온 그 순간에 막힌다.

마지막으로 `buff/`.

```
root@lumen-territory:/sys/magic# cat buff/strength_up.mag
```

```
function cast_strength_up(target) {
    base = calculate_base_power(target);      // 무거운 연산
    bonus = calculate_base_power(target) * 0.1;
    final = calculate_base_power(target) + bonus;
    refresh = calculate_base_power(target);   // 또?
    apply_buff(target, calculate_base_power(target) + bonus);
}
```

"……이건 그냥 멍청한 거잖아."

`calculate_base_power`. 대상의 기본 전투력을 계산하는, 가장 무겁고 오래 걸리는 연산이다. 그런데 이 코드는 그 무거운 연산을 한 번 호출하면 될 걸 **다섯 번이나** 호출한다. 똑같은 값이 나올 게 뻔한데도, 매번 처음부터 다시 계산한다.

캐싱이라는 개념을 모르는 거다. 한 번 계산했으면 변수에 저장해두고 갖다 쓰면 되는데, 이 코드는 그냥 필요할 때마다 매번 풀스캔을 돌린다. 다섯 번. 마나를 다섯 배로 태우면서.

이런 코드를 짜는 신입한테 내가 늘 하던 말이 있다. "넌 지금 같은 피자를 다섯 번 시켜놓고 한 조각씩만 먹고 버리는 거야." 그러면 신입은 어김없이 "그래도 결과는 똑같이 나오는데요?"라고 대꾸했다. 그래, 결과는 똑같이 나온다. 마나를 다섯 배로 태우고, 시전 속도가 다섯 배로 느려진다는 것만 빼면. 이 세계의 버프 마법사들이 한 명한테 힘 좀 실어주려고 헉헉대며 땀을 뻘뻘 흘리던 게, 다 이 멍청한 중복 연산 때문이었던 거다.

```
function cast_strength_up(target) {
    let base = calculate_base_power(target);   // 딱 한 번만
    apply_buff(target, base * 1.1);
    release_temp();
}
```

나는 의자에 등을 기대며 길게 숨을 내쉬었다.

화염, 힐, 보호막, 버프. 네 개를 까봤는데 네 개가 전부 똥코드였다. 메모리 릭, 의미 없는 `sleep`, 중복 연산. 신입 개발자가 짜도 이것보단 잘 짜겠다 싶은 코드가, 이 세계에선 "고대의 신성한 마법"이라는 이름으로 312년째 떠받들어지고 있었다.

"전부 똥코드잖아."

나는 헛웃음을 흘렸다. 그리고 그 순간, 머릿속에서 무언가가 딸깍 맞아떨어졌다.

[안티-레거시 체질].

이 몸뚱이가 평생 짊어진 저주. 마나 친화력 0. 마법 명가의 후손이면서 불씨 하나 못 피우던 폐급. 가문의 수치라 불리던 이유.

나는 지금까지 그게 '결함'인 줄 알았다. 마나를 못 다루는 고장 난 몸. 그런데 아니었다.

생각해보면 이상했다. 이 세계의 모든 영창은 — 방금 본 그 코드들처럼 — 의미 없는 `sleep`과 무한 루프와 중복 연산으로 떡칠된 비효율 덩어리다. 마법을 쓰려면 그 똥코드를 몸으로 받아들이고, 그대로 실행해야 한다. 영창을 읊고, 정령을 부르고, 3초를 기다리고, 9999번 루프를 돌리는 그 과정을.

그런데 아르시안의 몸은, 그걸 **거부했다.**

비효율적인 연산을 본능적으로 토해냈다. 의미 없는 `sleep`을 몸이 거절했다. 9999번 도는 무한 루프를 받아들이지 못했다. 그래서 마법이 안 됐던 거다. 마나가 부족해서가 아니라, 이 몸이 **더러운 코드를 컴파일하기를 거부하는 체질**이었기 때문에.

"저주가 아니었어."

나는 천천히 내 손바닥을 내려다봤다.

"이거…… 린터(linter)잖아."

코드에 더러운 게 있으면 빨간 줄을 쫙 그어버리는 그 깐깐한 검사기. 들여쓰기 하나 틀려도, 안 쓰는 변수 하나 있어도 경고를 토하며 빌드를 거부하는 그 깐깐한 놈. 아르시안의 몸이 딱 그거였다.

이 몸은 312년 묵은 레거시 똥코드를 본능적으로 린팅하고 있었던 거다. "이 코드는 비효율적입니다. 컴파일을 거부합니다." 그래서 마나 친화력 0. 다른 마법사들이 똥코드를 군말 없이 꾸역꾸역 실행하는 동안, 이 몸은 홀로 "그딴 더러운 코드는 못 돌린다"며 버티고 있었던 거다.

남들이 보기엔 폐급. 하지만 본질은, **이 세계에서 가장 코드 품질에 깐깐한 몸뚱이.**

그러니 root 권한이 깔끔하게 리팩토링된 3줄짜리 `ignite.mag`를 만나자마자, 이 몸은 군말 없이 받아들였다. 더럽지 않으니까. 통과 기준을 충족했으니까. 폐급이 하루아침에 무영창 먼치킨이 된 게 아니었다. 원래부터 깐깐한 검수자였던 몸에, 드디어 검수를 통과하는 깨끗한 코드를 먹여준 것뿐이었다.

소름이 돋았다. 동시에 통쾌했다.

8년을 코드 리뷰로 갈려나가며, "이딴 걸 코드라고 짰냐"고 후배들 멱살을 잡던 그 까칠함이, 죽어서 이세계까지 따라온 거다. 전생의 직업병이 현생의 체질이 됐다. 신이 장난친 건지 우연인지 몰라도, 나는 이 세계에 떨어질 운명이었던 것 같았다. 더러운 코드를 못 견디는 놈이, 더러운 코드로 가득 찬 세계로.

"하, 진짜……."

그때 문이 빼꼼 열렸다.

"도련님? 점심 식사…… 어, 또 그 까만 창 보고 계세요?"

마리였다. 쟁반을 들고 들어온 그녀는 내가 허공의 터미널을 들여다보는 걸 보더니 고개를 갸웃했다. 어제 뒷마당 폭발 이후, 마리는 내가 가끔 허공을 노려보며 손가락을 까딱이는 걸 봐도 더는 묻지 않았다. 그냥 "도련님이 또 이상한 거 하시네" 정도로 받아들이기로 한 모양이었다.

"마리. 좋은 소식 있어. 이제 사제들이 힐 좀 썼다고 안 죽어도 돼."

"네? 사제님들이…… 안 돌아가신다고요? 어떻게요?"

"메모리 릭을 잡았거든. `release` 한 줄이 빠져 있었어."

"리…… 리스? 그게 뭔데요? 새로 기르시는 강아지 이름인가요?"

"…………"

"릴리스라는 강아지가 힐을 한다고요? 마법 쓰는 강아지예요?! 세상에, 그런 신비한 동물이!"

마리의 눈이 초롱초롱해졌다. 머릿속에서 이미 마법 쓰는 강아지 한 마리가 폴짝폴짝 뛰어다니는 게 보이는 듯했다. 나는 잠시 정정할까 고민하다가, 그냥 포기했다. 어제는 '빌드 님'이라는 용병이 다녀갔고, 오늘은 '릴리스'라는 힐링 강아지가 생겼다. 이세계의 번역기는 늘 이런 식이었다.

"그래. 아주 착한 강아지야."

"역시! 도련님은 동물도 잘 다루시는군요!"

마리가 쟁반을 내려놓으며 환하게 웃었다. 나는 식은 수프를 떠먹으며 다시 터미널로 시선을 돌렸다.

마법 코드 네 개를 고쳤다. 화염, 힐, 보호막, 버프. 하지만 이건 시작에 불과했다. `/sys/magic/` 디렉토리엔 아직 열어보지도 않은 마법이 수십 개고, 그게 전부 똥코드일 게 뻔했다.

그런데 문득, 더 근본적인 질문이 떠올랐다.

마법이 이 꼴이라면, 영지는?

```
root@lumen-territory:~# ls -al /sys/territory/
drwxr-xr-x  irrigation/      # 관개
drwxr-xr-x  agriculture/     # 농업
drwxr-xr-x  water_supply/    # 식수
drwxr-xr-x  defense/         # 방어
drwxr-xr-x  population/      # 인구
drwxr-xr-x  finance/         # 재정
-rw-r--r--  config.broken    (size: 8.2MB, errors: 1043)
```

`config.broken`. 설정 파일 이름에 대놓고 'broken'이 박혀 있었다. 에러가 1043개. 8.2메가짜리 설정 파일에 에러가 천 개 넘게 쌓여 있다는 건, 이 영지가 지금 이 순간에도 콘솔에 빨간 에러 로그를 폭포처럼 토해내면서 간신히 굴러가고 있다는 뜻이었다.

빚, 흉작, 떠나는 영지민. 아버지가 평생 무력하게 끌어안고 있던 그 모든 문제들. 사람들은 그걸 '운이 없다'거나 '저주받은 땅'이라고 불렀을 거다.

하지만 나는 안다. 운도, 저주도 아니다.

**전부 똥코드다.**

마법이 똥코드였듯이, 이 영지도 어딘가 코드가 더럽게 짜여 있고, 어딘가에서 자원이 새고 있고, 어딘가에 의미 없는 `sleep`이 박혀서 모든 걸 멈춰 세우고 있는 게 분명했다. 그리고 그걸 까볼 권한을 가진 건, 이 세계에 나 하나뿐이었다.

나는 수프 그릇을 옆으로 밀어두고, 손가락을 가볍게 풀었다. 어제 화염 마법을 까볼 때처럼, 손끝이 근질거리기 시작했다.

"릴리스 강아지는 됐고."

"네? 도련님?"

"이제 영지를 까볼 시간이거든."

커서가 깜빡였다. `config.broken`이라는 빨간 글씨가, 마치 "어디 한번 열어보시지"라며 나를 도발하는 것 같았다.

좋다. 312년 묵은 영지 인프라. 안에 어떤 끔찍한 게 들어 있을지, 솔직히 좀 기대됐다.

```
root@lumen-territory:~# cd /sys/territory/
root@lumen-territory:/sys/territory# cat config.broken | head -20
```
