# 제 8화: cron으로 굴러가는 영지

[← 이전: 7화: 적자 영지의 영주](./ep007.md) | [시리즈홈](../README.md) | [목차](./README.md) | [다음: 9화: 몬스터 웨이브가 온다 →](./ep009.md)

영지를 고치겠다고 호기롭게 선언은 했는데, 막상 `/sys/territory/` 디렉토리를 까보니 한숨부터 나왔다.

```
root@lumen-territory:~# ls -al /sys/territory/
drwxr-xr-x  agriculture/
drwxr-xr-x  water/
drwxr-xr-x  defense/
drwxr-xr-x  trade/
-rw-r--r--  README.md          (last modified: 87 years ago)
```

87년 전. README 마지막 수정일이 87년 전이었다.

전생에서도 이런 코드 많이 봤다. 작성자는 이미 퇴사했고, 인수인계 문서는 없고, 주석이라곤 "// 일단 돌아감 - 건들지 마"가 전부인 코드. 그걸 건드리는 순간 회사 전체가 불타는 그런 코드. 루멘 영지가 딱 그 꼴이었다. 87년 동안 아무도 손 안 댄 레거시 위에서, 영지민들이 매일 손으로 마법을 영창해가며 간신히 굴리고 있었던 거다.

"농업이랑 수자원부터."

빚이고 흉작이고, 결국 근본은 먹고사는 문제다. 밭에 물이 안 돌고 작물이 죽어 나가니까 세금이 안 걷히고, 세금이 안 걷히니까 빚이 쌓이고, 빚이 쌓이니까 영지민이 떠난다. 전형적인 연쇄 장애다. 가장 아래 깔린 인프라 한 줄이 막히면, 그 위에 쌓인 모든 게 와르르 무너진다.

나는 `water/` 디렉토리부터 열었다.

```
root@lumen-territory:~/sys/territory/water# ls
irrigation.mag   well_pump.mag   reservoir.cfg   .DS_Store
```

"……`.DS_Store`는 또 왜 여기 있어."

창조신 새끼, 맥 쓰나 보다. 87년 전부터 이세계에 `.DS_Store`를 흘리고 다니는 창조신이라니. Gemini라는 이름값 한번 거하게 한다. 나는 일단 그건 무시하고 `irrigation.mag`를 열었다.

```
root@lumen-territory:~/sys/territory/water# cat irrigation.mag

function irrigate_field(field_id, mana) {
    summon_water_spirit();        // 정령 소환 (매번)
    wait_for_spirit_mood(random); // 정령 기분 맞추기???
    if (spirit.is_happy == true) {
        for (i=0; i<field.tiles; i++) {
            pour_water(tile[i], 1); // 한 칸에 한 방울씩
            sleep(500);
        }
    } else {
        return ERROR_SPIRIT_GRUMPY; // 정령이 삐짐
    }
}
```

나는 한참을 그 코드를 노려봤다.

물 한 번 대려고 매번 정령을 소환한다. 그 정령의 '기분'을 랜덤으로 맞춰야 하고, 기분이 안 좋으면 그냥 `ERROR_SPIRIT_GRUMPY`를 뱉고 종료된다. 운 좋게 정령이 기분이 좋아도, 밭 한 칸에 물 한 방울씩 부으면서 매번 0.5초씩 쉰다. 밭이 천 칸이면 천 칸을 하나하나 도는데, 칸마다 0.5초. 산수해보면 이 밭 하나 적시는 데만 8분이 넘게 걸린다는 소리다. 그것도 정령님 심기가 평안하셔야.

"시발, 이게 왜 안 됐는지 알겠네."

영지의 물 사정이 개판이었던 이유가 코드 한 방에 설명됐다. 비결정적 의존성(삐지는 정령), 동기 블로킹 루프(한 방울씩), 의미 없는 `sleep`. 디버깅 면접에서 "이 코드의 문제점을 모두 찾으시오"로 내면 지원자 거를 용도로 딱이다.

나는 소매를 걷었다. 손가락이 다시 근질거리기 시작했다.

```
root@lumen-territory:~/sys/territory/water# vi irrigation.mag
```

정령 의존성부터 들어냈다. 애초에 물 대는 데 왜 정령 기분이 필요한가. 그냥 수원지에서 물을 끌어다 비동기로 뿌리면 된다. 한 칸씩 도는 동기 루프는 `Promise.all`로 전체 타일에 동시에 살포하도록 바꿨다. `sleep(500)`은 당연히 삭제. 정령이 삐져서 작업이 통째로 죽는 그 미친 에러 핸들링도 갈아엎었다.

```
root@lumen-territory:~/sys/territory/water# cat irrigation.mag

async function irrigate_field(field_id) {
    const tiles = field[field_id].tiles;
    await Promise.all(
        tiles.map(t => pour_water(t, optimal_amount))
    );
    return OK;
}
```

8분짜리가 0.3초짜리가 됐다. 정령은 실업자가 됐고.

하지만 이걸로 끝내면 8년차 짬밥이 아깝다. 진짜 문제는 따로 있었다. 코드를 고쳐봤자, 이걸 매번 사람이 손으로 영창해서 실행해야 한다는 거. 영지에는 '물 영창사'라는 직책이 따로 있었는데, 그 사람이 매일 새벽에 일어나 밭마다 돌면서 영창을 외고 다닌다고 했다. 그게 그 사람 하루 일과의 전부였다.

전생의 나라면 이런 걸 보고 딱 한마디 했을 거다.

"이걸 왜 사람이 해?"

자동화하면 되잖아. 매일 같은 시간에 같은 작업을 반복하는 거? 그거 컴퓨터가 제일 잘하는 거다. 인간은 까먹고, 늦잠 자고, 아프고, 짜증 내지만, 스케줄러는 안 그런다. 정해진 시각에, 정확하게, 불평 한마디 없이 돈다.

나는 새 파일을 열었다.

```
root@lumen-territory:~/sys/territory# crontab -e

# m  h   task
  0  5   * * *   ./water/irrigate_all.mag        # 매일 새벽 5시 전 밭 관개
  0  *   * * *   ./water/well_pump.mag --refill  # 매시 정각 우물 자동 충수
  30 6   * * *   ./agriculture/growth_boost.mag  # 매일 6시 30분 작물 생장 촉진
  0  20  * * *   ./agriculture/soil_check.mag    # 매일 저녁 8시 토양 상태 점검
```

크론탭. 정해진 시각마다 마법 스크립트가 알아서 돌게 만드는 거다. 사람이 새벽에 일어날 필요도, 영창을 욀 필요도, 까먹을 일도 없다. 새벽 5시가 되면 시스템이 알아서 전 밭에 물을 대고, 매시 정각마다 우물을 채우고, 아침 6시 반엔 작물 생장을 촉진하고, 저녁 8시엔 토양 상태를 점검한다.

이 영지는 이제 사람이 아니라 cron으로 굴러간다.

`well_pump.mag`랑 작물 생장 스크립트도 마저 손봤다. 우물 펌프는 마나 누수가 30%나 있었고 — 힐링 마법에서 본 그 메모리 릭 패턴이랑 똑같았다 — 작물 생장 마법은 아예 자정에만 동작하도록 하드코딩되어 있어서 낮에 햇빛 받을 때 정작 안 돌고 있었다. 작성자가 시차 계산을 틀린 거다. 87년 동안 작물들은 한밤중에만 생장 촉진을 받고 있었던 셈이다. 그러니 크지.

전부 패치하고, 마지막으로 엔터를 쳤다.

```
root@lumen-territory:~/sys/territory# crontab -l && systemctl start territory-scheduler
[OK] crontab installed. 4 jobs scheduled.
[SYSTEM] territory-scheduler: ACTIVE
[CRON] triggering manual run: irrigate_all.mag ...
```

수동으로 첫 실행을 한 번 걸어줬다. 24시간 기다리며 검증할 순 없으니까. 스테이징에서 한 번은 돌려봐야 마음이 놓이는 법이다.

그리고 나는 창밖을 봤다.

처음엔 아무 일도 없었다. ep005 때랑 똑같이, 너무 조용해서 오히려 불안했다. 그러다—

땅이 젖기 시작했다.

영지 동쪽, 갈라질 대로 갈라져 거북 등껍질처럼 쩍쩍 금이 가 있던 밭. 그 마른 흙 위로 물줄기가 사방에서 솟아오르더니, 천 칸이 넘는 밭이 동시에, 일제히 검게 젖어 들어갔다. 한 방울씩이 아니라 전부 한꺼번에. 정령 기분 따위 묻지도 않고. 0.3초 만에 영지 동쪽 평야 전체가 거울처럼 물을 머금고 번들거렸다.

그게 끝이 아니었다.

새벽 6시 반 작업이 — 내가 수동으로 같이 걸어놨다 — 발동하면서, 물을 먹은 밭에서 시들어 누렇게 죽어가던 작물들이 거짓말처럼 고개를 들기 시작했다. 축 늘어졌던 밀 줄기가 빳빳하게 일어서고, 말라비틀어진 잎사귀에 푸른 기가 돌았다. 마치 빨리감기 영상처럼, 죽기 직전이던 밭 하나가 눈앞에서 살아났다.

"……오케이. 배포 성공."

나는 의자에 등을 기댔다. 이번엔 담장도 안 무너졌고, 화로도 안 녹았고, 누구 속옷도 안 탔다. 그냥 깔끔하게, 밭에 물이 돌았다. 모름지기 좋은 배포란 이래야 한다. 아무 일도 안 일어난 것처럼 조용히, 그러나 모든 게 달라지는. 사용자는 서버 안에서 무슨 일이 벌어지는지 알 필요가 없다. 그냥 어제 안 되던 게 오늘 되면, 그걸로 충분하다. 죽어가던 밭이 살아나는 데에 거창한 영창이나 정령님의 변덕은 필요 없었다. 필요한 건 제대로 짠 코드 한 덩이와, 그걸 정해진 시간에 돌려줄 스케줄러 하나뿐이었다.

문제는 그 '조용함'이 오래가지 않았다는 거다.

**우당탕탕탕!!**

또 그 발소리. 이젠 익숙하다.

"도, 도련님!!! 밭이!! 밭이 살아났어요!!"

마리가 문을 박차고 뛰어 들어왔다. 이번엔 우는 게 아니라 들떠서 비명을 지르고 있었다. 두 손을 마구 흔들면서.

"동쪽 밭이요!! 갑자기!! 물이 펑펑 솟더니!! 죽었던 밀이 막!! 막 일어났어요!! 제 눈으로 봤어요!!"

"응, 봤어. 내가 한 거야."

"네에?! 또 빌드 님이……?!"

"아니, 이번엔 빌드 님 안 오셨어."

지난번 '빌드 님' 거짓말의 여파가 아직도 굴러다니고 있었다. 기술 부채는 코드에만 쌓이는 게 아니라는 걸 절감하는 순간이었다.

"그, 그럼 어떻게……!"

"크론으로 돌렸어."

"크……론이요?"

마리가 고개를 갸웃했다. 또 시작이다.

"크론 님은…… 또 어떤 분이신가요?"

"……사람 아니야."

"그럼 정령님이세요?! 새로운 물의 정령님을 부르신 거예요?!"

"그것도 아니고."

설명을 포기할까 하다가, 이번엔 좀 해주기로 했다. 어차피 앞으로 영지 시스템은 죄다 이걸로 돌릴 거고, 마리는 메이드장 비슷한 위치라 알아두면 나쁠 게 없었다.

"마리. 매일 새벽마다 물 영창사 아저씨가 밭 돌면서 영창하잖아."

"네! 톰 아저씨요. 매일 새벽 4시에 일어나셔서……."

"이제 안 일어나도 돼. 새벽 5시 되면 영지가 알아서 물을 대니까."

"……영지가, 알아서요?"

"응. 매일. 정해진 시간에. 톰 아저씨 늦잠 자도, 아파도, 영지는 안 멈춰."

마리가 입을 반쯤 벌린 채로 한참을 멍하니 서 있었다. 그러다 천천히, 아주 천천히, 그 의미를 곱씹는 표정이 됐다. 외계어를 듣는 표정이 아니라 — 이번엔, 뭔가 무서운 걸 본 표정이었다.

"도련님. 그건…… 그건 마치, 영지가 살아 있는 것 같잖아요. 누가 시키지 않아도 스스로……."

"비슷해. 근데 살아 있는 건 아니고, 그냥 시간표대로 도는 거야."

"그게 더 무서운데요……."

이세계 메이드의 직감이란 게 가끔 무섭게 정확하다. 사람이 안 시켜도 스스로 돌아가는 시스템. 인간 없이 굴러가는 영지. 그게 경이로우면서 동시에 등골이 서늘한 거, 나도 안다. 전생에서 무인 자동화 파이프라인 처음 짠 날, 나도 똑같은 기분이었으니까. 내가 자는 동안에도 코드가 알아서 배포되고 있다는 거, 처음엔 든든하다가 어느 순간 좀 소름 끼친다.

그날 하루 만에 영지가 들썩였다.

소문은 물보다 빠르게 번졌다. 동쪽 밭이 하루아침에 살아났다는 얘기는 점심도 되기 전에 영지 전체에 퍼졌고, 오후가 되자 영지민들이 죽었다던 밭 앞에 모여 웅성거리기 시작했다. 어떤 노인은 거북 등딱지처럼 갈라졌던 흙이 까맣게 젖어 든 걸 손으로 만져보고는 성호를 그었고, 어떤 아낙은 다시 일어선 밀 줄기를 붙들고 울었다. 흉작 7년에 자식들 굶기던 사람들이었다.

그리고 그 시선들이, 하나둘 우리 성을 향하기 시작했다.

"……들었나? 도련님이 하셨다는데."

"그 도련님? 마나도 못 다룬다던 그……?"

"앓아눕고 나더니 사람이 달라졌다더라. 무영창으로 뒷마당을 통째로 날렸다는 얘기도 있고."

"그게 진짜였어?"

폐급 도련님. 가문의 수치. 불씨 하나 못 피우던 마법 명가의 망신. 그 호칭들이 그날을 기점으로 천천히 갈리기 시작했다. 영지민들의 눈빛 속에서, '저 모자란 도련님'이 '저 무서운 도련님'으로, 그리고 어쩌면 '저분이라면'으로 미세하게 업데이트되고 있었다. 평판이라는 건 코드 같아서, 한 줄 바꾼다고 바로 반영되진 않는다. 캐시가 남아 있으니까. 사람들 머릿속에 '저놈은 폐급'이라고 박힌 정보는 쉽게 안 지워진다. 하지만 한 번 결과를 눈으로 보여주면, 그 캐시는 통째로 무효화된다. 7년 만에 살아난 밭이 딱 그 역할을 했다. 백 마디 말보다, 까맣게 젖어 든 흙 한 줌이 더 강한 증거였다.

그날, 첫 커밋이 머지됐다. 내 첫 번째 치적이었다.

저녁 무렵엔 아버지까지 동쪽 밭에 다녀왔다. 루멘 남작은 흙냄새와 물기를 잔뜩 묻힌 채로 돌아와, 한참을 말없이 나를 봤다. 그 복잡한 눈빛이 지난번보다 공포는 좀 줄고 감격은 좀 늘어 있었다.

"아르시안. 그 밭은…… 7년째 한 톨도 안 나던 땅이다."

"이제 날 거예요."

"그게…… 정말이냐."

"매일 새벽에 물 대고, 아침에 생장 촉진하고, 저녁에 토양 점검합니다. 사람 손 안 거치고요. 비 안 와도, 영창사가 아파도, 안 멈춰요."

아버지는 한동안 대답을 못 했다. 그러다 굽었던 어깨를 아주 미세하게 폈다. 평생 빚과 흉작에 짓눌려 있던 그 어깨가, 처음으로 조금 가벼워 보였다.

"……고맙다."

짧은 한마디였지만, 나는 그게 ep005 때 차마 못 한 사과의 다른 버전이라는 걸 알았다. 츤데레 기질은 부자가 닮나 보다. 나는 굳이 아는 척하지 않고 고개만 끄덕였다.

방으로 돌아와 터미널을 띄웠다. `territory-scheduler`는 백그라운드에서 조용히 돌고 있었다. 내가 자는 동안에도, 내일도, 모레도, 이 영지는 시간표대로 자기 자신을 돌볼 거다. 사람의 게으름이나 망각에 의존하지 않고.

```
root@lumen-territory:~# systemctl status territory-scheduler
● territory-scheduler - Lumen Territory Auto Ops
   Active: active (running)
   Jobs: 4 scheduled, 0 failed
   [INFO] next run: irrigate_all.mag @ 05:00
```

failed 0. 깨끗했다.

나는 의자에 깊숙이 몸을 묻고, 오랜만에 좀 뿌듯한 기분을 느꼈다. 8년간 남의 회사 서버 고쳐주다 과로사한 인간이, 이세계 와서야 처음으로 '내 인프라'를 고치고 있었다. 그것도 누구 눈치 안 보고, 코드 리뷰 반려당할 일도 없이, root 권한으로. 솔직히 말하면, 죽기 전보다 지금이 더 살 만했다.

다음은 뭘 고칠까. 무역 디렉토리도 똥덩어리던데. 트레이드 라우팅 한번 까보면—

**똑똑똑.**

노크 소리. 마리였다. 그런데 들떠 있던 아까랑 톤이 달랐다. 목소리가 잔뜩 죽어 있었다.

"도련님…… 잠깐 괜찮으세요?"

"들어와."

문을 열고 들어온 마리의 얼굴에 핏기가 없었다. 손에는 흙 묻은 쪽지 한 장이 들려 있었다. 영지 외곽 초소에서 올라온 거라고 했다.

"방금…… 서쪽 초소에서 전령이 왔는데요."

"무슨 일인데."

"어둠의 숲에서…… 몬스터들이 떼로 움직이고 있대요. 그것도 한두 마리가 아니라…… 본 적 없는 숫자라고. 사냥꾼들이 다 도망쳐 왔어요."

나는 손에서 멈췄다.

마리가 떨리는 목소리로 마지막 한 줄을 읽었다.

"'웨이브가 온다'고…… 그렇게 전하라고 했답니다."

밭에 막 물이 돌기 시작한 영지. 7년 만에 처음 고개를 든 밀밭. 첫 치적에 들뜬 영지민들. 그 위로, 어둠의 숲이 검은 그림자를 드리우고 있었다.

나는 천천히 터미널을 다시 끌어왔다. 커서가 자비 없이 깜빡였다.

```
root@lumen-territory:~# tail -f /var/log/monster_spawn.log
[WARN] dark_forest spawn rate: +1200% (last 6h)
[WARN] entity count exceeds threshold: estimating...
[ALERT] incoming wave detected.
```

"……하. 운영 들어가니까 바로 트래픽 폭주네."

배포 끝나고 한숨 돌리려던 참에 터지는 장애. 이것도 어쩐지, 전생이랑 똑같았다.
