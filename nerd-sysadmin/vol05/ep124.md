# 제 124화: 제국 패치 노트

[← 이전: 123화: 신앙 API](./ep123.md) | [시리즈홈](../README.md) | [목차](./README.md) | [다음: 125화: 서드파티 경고 →](./ep125.md)

5권, 제국 네트워크 병합 및 분산 처리. 통일 네트워크의 결말 구간이었다.

베르단이 코드를 읽기 시작하면서 제국 쪽은 빠르게 가라앉았다. 정작 시끄러워진 건 우리 편이었다. 승리에 취한 왕국 귀족들이 제국을 '점령지'라 부르며, 제국민에게는 더 높은 세금과 더 낮은 권한을 매기자고 몰려왔다. "이긴 쪽이 더 가져야 하지 않습니까." 한 귀족이 당연하다는 듯 말했다.

나는 정복을 선언할 생각이 없었다. 정복은 결국 또 다른 단일 지배자를 세우는 일이고, 그건 방금 부순 황제 자리를 이름만 바꿔 다시 만드는 짓이다. 통합은 한쪽이 다른 쪽을 밟고 올라서는 게 아니다. 같은 규칙이 양쪽에 똑같이 적용될 때, 비로소 둘은 한 시스템이 된다. 한쪽에만 특혜를 박아 넣은 권한 테이블은, 4권에서 걷어낸 그 썩은 귀족 특권과 정확히 같은 길을 간다.

그래서 나는 정복 칙령 대신 패치 노트를 띄웠다.

```
root@lumen-control:~# ./publish_release_note.sh --version continent-1.0
[SYSTEM] 양국 모두에 같은 권한 규칙이 적용된다.
```

대륙 1.0. 왕국이든 제국이든 같은 인증을 거치고, 같은 권한 규칙을 따른다. 이긴 쪽이라 해서 더 받는 예외 따위는 없었다. 귀족들의 얼굴이 일그러졌지만, 패치 노트는 협상문이 아니라 배포 기록이다. 이미 적용돼 돌아가는 규칙을, 마음에 안 든다고 누가 되돌릴 수는 없다. 엘리스가 조용히 덧붙였다. "이건 선언이 아니라, 그냥 사실을 적은 거네요." 맞다. 좋은 시스템은 누가 이겼는지 묻지 않는다. 누구에게 권한이 있는지만 묻는다. 판교에서 겪은 일도 다르지 않았다. 프로젝트가 성공할 때마다 누가 더 기여했네 하며 권한을 더 달라는 사람이 꼭 나왔지만, 오래 버티는 조직은 기여도 순이 아니라 책임 범위 순으로 권한을 나눴다.

그래서 나는 발표가 끝났다는 말 대신, 로그부터 저장했다.

```
root@lumen-control:~# printf '%s' '제국 패치 노트' >> /incident/vol05.log
[SYSTEM] 인시던트 로그가 저장되었습니다.
```

대륙은 이제 같은 규칙 위에서 돌았다. 사람과 사람 사이의 전쟁은, 이걸로 한 매듭이 지어진 셈이었다. 문제는 이 대륙이 세계의 전부가 아니라는 데 있었다.

그때, 드워프와 엘프 사절단이 동시에 긴급 방문을 요청한다.

나는 콘솔을 접지 않았다. 인간들의 네트워크는 한 규칙으로 묶었지만, 산맥의 드워프도 숲의 엘프도 저마다 다른 규격 위에서 돌아가고 있었다. 우리가 아직 한 번도 손대 본 적 없는 시스템이었고, 다음 로그는 거기서 시작될 터였다.
