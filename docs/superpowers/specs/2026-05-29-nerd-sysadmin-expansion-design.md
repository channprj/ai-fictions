# 설계 스펙 — nerd-sysadmin 10권 라이트노벨 확장

- 날짜: 2026-05-29
- 상태: 1권 파일럿 진행 중

## 목표

`nerd-sysadmin`을 권당 약 200페이지(≈12만 자) × 10권(≈120만 자)의 라이트노벨로 확장한다. 성인 웹소설 톤(빠른 사이다·먼치킨·IT밈·가벼운 욕설).

## 확정 결정 (사용자)

1. **캐던**: 1권을 먼저 완성(파일럿)해 품질·포맷 기준을 확정한 뒤 2~10권 진행 방식 결정.
2. **권 내부 구조**: 회차 연재형 — 권별 폴더 + `ep{NNN}.md`(회차 ~5,000자, 권당 25화), 회차 번호는 권을 가로질러 연속(1권 = ep001~025).
3. **톤**: 기존 프롤로그·1장의 성인 웹소설 톤 유지.
4. **정리**: 루트의 `writer*.py` 생성 스크립트 삭제(+ 히스토리 제거 예정), dist 스텁 ~210개 삭제(dist는 배포 zip 전용).
5. **정식 본문 소스**: 기존 최상위 13개 파일을 `outline/`의 마스터 시놉시스로 보존하고, 이를 확장해 실제 회차를 집필.

## 디렉토리 구조

```
nerd-sysadmin/
├── README.md            # 시리즈 개요
├── BIBLE.md             # 시리즈 바이블(캐릭터/시스템/IT밈/메타플롯)
├── outline/             # 마스터 시놉시스 + 1권 비트시트 + 기존 집필 시드
│   ├── 00~12 (*.md)     # 권별 시놉시스/프로즈
│   ├── vol01-beats.md   # 1권 25화 비트시트
│   └── seed-episodes/   # 기존 38화 드래프트(캐논 시드)
├── vol01/               # 파일럿 집필 결과물
│   ├── README.md        # 1권 회차 목차
│   └── ep001.md … ep025.md
└── dist/                # 배포 zip (집필 완료 후)
```

## 집필 방식

- 서브에이전트가 `BIBLE.md` + `outline/vol01-beats.md` + 해당 화 시드를 읽고 회차를 직접 집필(파일 작성).
- 메인 세션이 검수·연속성 보정·커밋.
- 1권 25화 완성 후 `vol01/README.md`·시리즈 `README.md`·dist zip 생성 → 사용자 리뷰 → 2~10권 캐던 결정.

## 미결/후속

- `writer*.py` git 히스토리 제거(`git filter-repo`)는 로컬 수행 후 원격 force-push가 필요 — 별도 확인.
- 2~10권 비트시트는 1권 리뷰 후 동일 방식으로 작성.
