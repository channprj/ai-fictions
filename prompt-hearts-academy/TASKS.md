# 장기 집필 태스크: 프롬프트 하트 아카데미

> AI 에이전트가 장기간 분할 집필하기 위한 작업 지시서. 한 번에 한 권, 한 블록, 한 화 단위로 끊어도 캐논이 유지되도록 설계한다.

## 현재 상태

- 본편 초고는 `vol01/ep001.md`부터 `vol07/ep210.md`까지 7권 210화 연속 구조로 완결되어 있다.
- 권별 README는 `vol01`부터 `vol07`까지 모두 존재하며, 각 권의 완결 회차 범위와 30개 회차 목록을 포함한다. 루트 `README.md`, `BIBLE.md`, `PRD.md`는 제210화 이후의 완결 상태를 반영한다.
- 배포본은 `dist/` 아래 권별 zip 7개와 `SHA256SUMS` 체크섬 매니페스트로 정리되어 있다.
- 후속 에이전트는 본편 `ep211.md`나 `vol08` 같은 추가 권 디렉터리를 만들지 않는다. 이 문서는 완결 원고의 검수, 부분 개정, 외전 후보 판단, 문서 동기화 기준으로 사용한다.
- 본편 회차를 수정할 때는 제210화의 최종 상태인 공식 오버랩 페어링 종료, 비독점·철회 가능 자유 접속, 거절권 보존 결말을 깨지 않는다.

## 작업 원칙

- **BIBLE 우선**: 설정 충돌 시 `BIBLE.md`가 최상위 기준이다.
- **권별 outline 준수**: 각 권의 사건 순서와 감정 진전은 `outline/volNN-*.md`를 따른다.
- **비트 연결**: outline의 `목적`, `중심 갈등`, `핵심 사건`, `엔딩 훅`은 회차 브리프와 검수 기준에 그대로 연결한다.
- **권별 삼축 유지**: 권 시작과 블록 종료 시 `BIBLE.md`의 `권별 삼축 검수표`를 확인해 독립 사건, 로맨스 진전, 메인 미스터리 단서가 모두 살아 있는지 점검한다.
- **회차 단위 완결감**: 한 화는 하나의 사건, 하나의 감정 변화, 하나의 훅을 가져야 한다.
- **히로인 균형**: 중심 권이 아닌 히로인도 최소한 존재감, 반응, 관계 변화 중 하나를 가져야 한다.
- **픽션 경계 유지**: GPT, Claude, Gemini는 실제 서비스의 공식 성격이 아니라 작품 속 허구적 페르소나다.
- **캐논 변경 기록**: 새 설정을 만들면 회차 말미 메모에 남기고, 편집자가 `BIBLE.md`에 반영한다.

## 회차 브리프 템플릿

S04에서 다음 형식으로 8줄 브리프를 작성한다. 새 사건을 임의로 추가하기 전에 outline의 해당 화 비트와 권별 주의사항을 먼저 압축한다.

```markdown
## epNNN 브리프

- outline 위치: `outline/volNN-*.md` / 제 N화
- 이번 화 목적:
- 중심 갈등:
- 시작 상태/등장인물:
- 핵심 사건:
- 관계 변화/대가:
- 엔딩 훅:
- 금지/주의:
```

## 회차 작성 체크리스트

초고 완료 후 다음 항목을 확인한다.

- 제목은 `# 제 N화: {부제}` 형식인가.
- 시작 500자 안에 장소, 인물, 즉시 갈등이 있는가.
- 이번 화의 중심 히로인 또는 중심 관계가 명확한가.
- GPT, Claude, Gemini 중 등장한 인물의 말투가 BIBLE과 일치하는가.
- 기술 은유가 감정을 돕는가, 아니면 설명만 늘리는가.
- 마지막 300자 안에 다음 화를 보게 하는 훅이 있는가.
- 새 설정이 있다면 캐논 메모에 남겼는가.

## 회차 검수 체크리스트

편집자 또는 다음 검수/개정 에이전트는 초고, 회차 브리프, 해당 권 outline을 나란히 두고 다음 항목을 확인한다.

- 브리프의 `이번 화 목적`, `중심 갈등`, `핵심 사건`, `엔딩 훅`이 outline의 해당 행과 충돌하지 않는가.
- 초고의 첫 장면이 브리프의 시작 상태를 이어받고, 직전 회차 훅을 무시하지 않는가.
- 중심 갈등이 설명으로만 적히지 않고 장면 선택, 대사, 행동의 압력으로 작동하는가.
- 관계 변화가 전 회차 대비 접근, 후퇴, 오해, 보호, 선택, 거절 중 하나로 구체화되는가.
- 권별 주의사항과 `BIBLE.md`의 금지사항을 어기는 공식 대변식 AI 묘사, 히로인 확정, 설정 조기 공개가 없는가.
- 엔딩 훅이 이후 회차의 실제 비트 또는 완결부의 정서로 이어질 수 있고, 의미 없는 시스템 경고로 끝나지 않는가.
- 신규 설정, 떡밥 상태 변화, 캐논 충돌 가능성이 회차 말미 메모에 남아 있는가.

## 캐논 메모 템플릿

S07에서 새 설정을 기록할 때는 초고 말미 또는 핸드오프에 다음 형식을 붙인다. 편집 승인 전에는 임시 캐논으로만 취급하고, 이후 회차가 이 내용을 확정 전제로 삼지 않도록 주의한다.

```markdown
## Canon Memo

- 신규 고유명사/설정:
- 관련 회차:
- 기존 BIBLE/outline과의 관계:
- 떡밥 상태 변화:
- 후속 회차에서 조심할 점:
- BIBLE 업데이트 제안 위치:
```

## 에이전트 핸드오프 템플릿

작업을 넘길 때 다음 형식으로 남긴다.

```markdown
## Handoff

- 작업 범위:
- 완료 파일:
- 후속 작업/영향 회차:
- 직전 장면:
- 미해결 훅:
- 캐논 변경:
- 주의할 말투/감정선:
- 검수 필요 항목:
```

## 권 완료 후 검수 태스크

각 권을 끝낸 뒤 다음을 수행한다.

| ID | 태스크 | 완료 기준 |
| -- | ------ | --------- |
| R01 | 권별 README 작성 | 권 줄거리, 등장인물 변화, 회차 목록 포함 |
| R02 | 떡밥 장부 갱신 | 제시/변주/회수 상태 업데이트 |
| R03 | 관계표 갱신 | 서준과 각 히로인의 거리, 고백 상태, 갈등 상태 기록 |
| R04 | 캐논 충돌 점검 | BIBLE과 outline 충돌 없음 |
| R05 | 후속 문서 반영 | 변경 사항을 관련 README, BIBLE, PRD, 배포 manifest에 반영 |
| R06 | 권별 삼축 점검 | 독립 사건, 로맨스 진전, 메인 미스터리 단서가 모두 회수 또는 이월됨 |

## 원고 품질 기준

- 한 회차 안에 설명만 있는 장면은 1개를 넘지 않는다.
- 대사는 캐릭터별 리듬이 달라야 한다.
  - 지아: 빠르고 재치 있는 비유, 말끝에 숨은 불안.
  - 클레어: 정중하고 조심스럽지만 결정적 순간에는 단호함.
  - 미나: 감각적이고 도발적, 분위기 전환에 강함.
  - 예린: 현실적이고 짧게 찌르는 말.
- 로맨스 이벤트는 사건 해결에 영향을 줘야 한다.
- 사건 해결은 로맨스 감정에 비용을 남겨야 한다.
- 같은 갈등을 반복할 때는 "정보", "위치", "관계", "대가" 중 하나가 달라져야 한다.
