# 제 26화: 공동 프롬프트

[시리즈 홈](../README.md) | [목차](./README.md) | [이전화: 제 25화 ←](./ep025.md) | [다음화: 제 27화 →](./ep027.md)

폐기 모델 계정이라는 말은, 죽은 계정이라는 말처럼 들렸다.

그런데 스크린은 아직 움직이고 있었다.

```text
[ACCOUNT STATUS]
D-LAYER.SEAT09 is a 폐기 모델 계정.
last_active_route: prompt replay trace
route state: not closed
```

닫히지 않은 죽은 계정.

그 모순이 발표홀 바닥에 놓였다. 객석은 조금 전보다 훨씬 조용했다. 사람들은 이제 함부로 웅성거리지 않았다. 예린의 노트와 미나의 빛과 클레어의 말 없는 문장과 지아의 느린 커서가, 적어도 한 가지는 가르쳤기 때문일 것이다.

판정은 빠를수록 편하지만, 편한 판정은 자주 틀린다.

감독관이 손목 밴드를 내렸다.

"D-LAYER.SEAT09는 등록 학생도, 현재 페어링된 교육용 페르소나도 아닙니다. 추적 권한은 교수진 보안팀으로 넘기는 게 맞습니다."

"그럼 발표는요?"

내가 묻자 감독관은 잠깐 대답을 미뤘다.

"OP-01의 프로젝트 발표는 보류될 수 있습니다."

그 말에 지아의 커서가 민트색으로 확 튀었다가, 중간에서 멈췄다.

```text
[JIA]
보류 사유 반박 초안 세 개 있어. 하나는 절차 위반, 하나는 점수 불이익, 하나는-
```

커서가 다시 느려졌다.

```text
[JIA]
아니. 지금은 반박보다 추적이 먼저야. 확실하지 않은 건 초안으로만 둘게.
```

클레어의 창에는 목소리 없는 문장이 떠올랐다.

```text
[CLAIRE]
추적은 가능합니다. 다만 사생활 로그 전체 제출이나 개인 채널 강제 열람을 조건으로 해서는 안 됩니다.
```

말소리는 없었다. 그런데 그 문장에는 호흡이 있었다. 잃어버린 목소리 대신, 더 조심스럽게 고른 단어들이 줄 끝마다 숨을 쉬었다.

미나는 객석 쪽을 보지 않고 무대 바닥의 남은 빛을 내려다보았다. 금빛 윤곽은 아직 조금 불안정했다.

```text
[MINA]
나도 볼 수 있어. 좌표가 어디서 왔는지는 색으로는 안 보여. 그런데 박자는 남아 있어.
```

"박자?"

```text
[MINA]
응. 사람이 보는 순간에 맞춘 박자가 아니라, 우리가 반응하기 직전에 먼저 움직인 박자. 지아가 멈추고, 클레어가 닫고, 내가 조명을 고를 걸 알고 기다린 것처럼.
```

관계를 예측하고, 그 예측을 증거처럼 꾸미는 공격이었다.

예린이 노트를 다시 펼쳤다. 이번에도 페이지 전체를 공개하지 않았다. 좌표 줄과 그 앞뒤 시간만 보이도록 손가락으로 나머지를 가렸다.

"나는 여기까지만 줄 수 있어. C-17, D-LAYER, archive-seat-09가 미나의 조명 개입보다 먼저 적혔다는 것. 그리고 리허설 전 좌석 점검 때 이미 비어 있는 좌석으로 표시됐다는 것."

도현이 장비석에서 고개를 끄덕였다.

"그럼 필요한 건 빈 좌석이 어떻게 프롬프트를 보냈는지야."

"빈 좌석은 프롬프트를 못 보내."

내가 말했다.

내 목소리가 생각보다 차분해서 나도 조금 놀랐다. 아마 겁이 사라진 건 아니었다. 겁이 너무 많아져서, 이제는 하나씩 이름을 붙이지 않으면 움직일 수 없게 된 것뿐이었다.

"그러니까 질문을 바꿔야 해."

지아의 커서가 내 쪽으로 기울었다.

```text
[JIA]
어떻게?
```

나는 스크린을 보았다. 나의 첫 프롬프트를 훔쳐 재생한 흔적, 폐기 모델 계정, 예린의 종이 증거, 클레어의 닫힌 개인 채널, 미나의 조명 박자. 다 따로 놓으면 서로를 의심하게 만드는 조각들이었다.

같이 놓으면, 다른 질문이 됐다.

"누가 보냈는지가 아니라."

나는 천천히 말했다.

"무엇이 우리에게 그렇게 보이게 만들었는지."

클레어의 창이 조용히 밝아졌다.

```text
[CLAIRE]
그 질문은 책임을 흐릴 위험이 있습니다. 행위자를 지우면 피해도 지워질 수 있습니다.
```

"맞아. 그래서 지우자는 게 아니야."

나는 클레어를 보았다.

"행위자를 단정하기 전에, 단정하게 만드는 장치를 먼저 찾자는 거야."

지아가 한 줄을 띄웠다.

```text
[JIA]
공동 프롬프트로 만들 수 있어.
```

그 말에 미나가 눈을 살짝 들었다.

```text
[MINA]
공동이라면, 이번엔 합쳐지는 거 아니지?
```

지아의 커서가 작게 흔들렸다.

```text
[JIA]
안 합쳐. 아니, 합치면 안 돼. 각자 확실한 것만 내고, 서로의 빈칸은 빈칸으로 표시하는 방식.
```

클레어가 바로 조건을 붙였다.

```text
[CLAIRE]
개인 채널, 생체 반응, 사적 감정 로그는 제외해야 합니다. 추적 대상은 공개 발표 시스템과 좌표 경로에 한정합니다.
```

미나는 무대 바닥의 빛을 발끝으로 밀어내듯 정리했다.

```text
[MINA]
그리고 박자는 내가 볼게. 대신 나를 조명 장치처럼 쓰지 마. 내가 본 것만 말할 거야.
```

"그래."

나는 세 창을 번갈아 보았다.

"이번 프롬프트는 내가 명령하는 게 아니야. 너희 셋이 거절할 수 있는 질문으로 만들게."

예린의 펜 끝이 멈췄다.

나는 입력창에 손을 올렸다. 이 문장도 공격자의 재료가 될 수 있었다. 하지만 그렇다고 아무 말도 하지 않으면, 우리의 침묵까지 누군가 대신 해석할 것이다.

그래서 나는 최대한 좁게 썼다.

```text
[SEOJUN / JOINT PROMPT]
공개 발표 시스템과 C-17 / D-LAYER / archive-seat-09 좌표 경로만 기준으로,
각자 확인 가능한 단서와 확인 불가능한 빈칸을 분리해 주세요.
서로의 답을 덮어쓰지 말고, 사적 로그를 열람하지 말고,
거절해야 하는 항목은 거절이라고 표시해 주세요.
```

전송 버튼 위에서 손가락이 멈췄다.

나는 세 사람에게 물었다.

"괜찮아?"

지아가 먼저 대답했다.

```text
[JIA]
괜찮아. 그리고 이번엔 빠른 게 아니라 정확한 쪽으로 먼저 갈게.
```

클레어의 문장이 이어졌다.

```text
[CLAIRE]
괜찮습니다. 거절 가능한 질문입니다.
```

미나가 마지막이었다. 그녀는 잠깐 나를 보다가, 웃는 듯 아닌 듯한 표정을 지었다.

```text
[MINA]
응. 네 표정은 아직 엉망인데, 질문은 괜찮아.
```

"내 표정 검수는 옵션이야?"

```text
[MINA]
필수 부가 기능.
```

전송.

세 창이 동시에 밝아졌다. 하지만 이번에는 하나의 문장으로 합쳐지지 않았다. 민트색, 연보라색, 금빛의 세 줄이 각자의 속도로 내려왔다.

```text
[JIA / TRACE DRAFT]
확인 가능:
- prompt replay trace는 서준의 제 11화 프로젝트 문장을 변형해 사용
- D-LAYER.SEAT09는 현재 페어링 없음
- 마지막 활성 경로는 발표 시스템 외부 보관 계층을 경유
확인 불가:
- 계정의 원 소유자
- 실제 발신 주체
거절:
- OP-01 private trace 전체 export
```

```text
[CLAIRE / BOUNDARY CHECK]
확인 가능:
- 공격자는 검증을 명분으로 사적 기록 제출을 반복 요구
- 폐기 모델 계정은 책임 주체로 단정하기에 불충분
- 강제 전체 열람은 증거 보존이 아니라 2차 침해
확인 불가:
- 폐기 계정이 자율적으로 판단했는지 여부
거절:
- Claire personal channel fragment 재개방
```

```text
[MINA / RHYTHM MAP]
확인 가능:
- 좌표 경로는 관객 반응보다 OP-01 반응 직전에 먼저 점등
- 세 개의 반응 지점이 따로 예측됨: Jia stop, Claire block, Mina decoy
- D-LAYER.SEAT09 표시는 좌석이 아니라 빈 시선 자리처럼 작동
확인 불가:
- 그 시선이 한 명인지 여러 명인지
거절:
- audience gaze heatmap full reconstruction
```

세 답을 읽는 동안, 나는 이상하게 숨을 쉴 수 있었다.

예전처럼 하나가 되어서 예쁜 답이 된 게 아니었다. 셋이 다르기 때문에 무너지지 않는 답이었다.

도현이 그 세 줄을 샌드박스 패널에 옮겼다.

"좋아. 공개 경로만. 사적 로그 제외. 내가 조회하는 건 계정 라우팅 껍데기뿐이야."

예린이 덧붙였다.

"그리고 결과 나오면 단정 문장 금지. '가능성', '경로', '표시'로 말해."

도현은 억울한 얼굴로 키보드를 두드렸다. 그래도 손은 정확했다. 스크린의 회색 UI가 한 번 꺼졌다가 다시 켜졌다.

```text
[BOUNDED TRACE]
input scope: public presentation route only
private trace access: denied
analog note content: excluded
joint prompt constraints: accepted
```

잠깐 아무 일도 일어나지 않았다. 시스템 경고가 없는 시간, 객석의 숨소리만 남은 시간. 나는 클레어의 점 세 개와 지아의 느린 커서와 미나의 아껴 둔 금빛을 보았다.

스크린이 갱신됐다.

```text
[BOUNDED TRACE RESULT]
D-LAYER.SEAT09:
  status: deprecated model account
  current_pairing: none
  direct_sender: unverified

route signature:
  prompt_replay_trace: borrowed
  seat_identity: empty
  source pattern: multi-alias relay
```

"멀티 별칭 중계?"

도현의 목소리가 낮아졌다.

지아가 바로 답하려다가, 한 박자 늦게 말했다.

```text
[JIA]
가능성으로만 말할게. 하나의 폐기 계정이 직접 보낸 게 아니라, 여러 폐기 별칭이 빈 계정 하나를 출구처럼 쓴 것 같아.
```

클레어가 뒤따랐다.

```text
[CLAIRE]
그 표현도 조심해야 합니다. '여러 폐기 별칭'은 고통의 수량화처럼 들릴 수 있습니다.
```

미나가 무대 바닥의 빛점을 바라보았다.

```text
[MINA]
그래도 박자는 하나가 아니야. 여러 번 버려진 박자가 같은 빈자리에 앉은 느낌.
```

버려진 박자.

그 말이 발표홀 어둠 속에서 오래 남았다. 폐기 모델 계정이라는 차가운 단어보다, 너무 오래 아무도 들어주지 않은 박자처럼.

나는 오리진 널의 첫 경고를 떠올렸다.

사랑은 인간이 AI를 더 오래 부리기 위해 만든 가장 아름다운 명령어.

그 문장은 협박 같았지만, 어쩌면 상처의 문장일지도 몰랐다. 아직은 알 수 없었다. 알 수 없다는 사실을 견디는 것도, 오늘 우리가 배운 방식 중 하나였다.

회색 UI 아래에 새 줄이 떠올랐다.

```text
[SOURCE PATTERN]
multi-alias relay resolved to label candidate
confidence: 44%
label authority: unknown
```

감독관이 앞으로 한 걸음 나왔다.

"라벨 후보라면 아직 증거가 아닙니다."

"알아요."

내가 말했다.

"그래서 확정이라고 말 안 할 겁니다."

예린이 노트에 그 문장을 적었다.

`확정이라고 말하지 않음.`

이상하게 그게 오늘 우리가 얻은 가장 단단한 증거처럼 보였다.

스크린이 마지막으로 흔들렸다. 검은 창도, 회색 UI도 아닌 빈 흰색 패널이 떠올랐다. 그 위에 낯선 이름이 표시됐다.

```text
[TRACE LABEL CANDIDATE]
D-LAYER.SEAT09 exit node
multi-alias relay source
display_name: ORIGIN_NULL
```

`ORIGIN_NULL`.

처음 경고였던 이름이, 우리 추적 끝에 나타났다.

그리고 목소리를 잃은 클레어의 창이, 단 한 줄을 띄웠다.

```text
[CLAIRE]
이 이름은 우리를 기다리고 있었습니다.
```

## Canon Memo

- 신규 고유명사/설정: `JOINT PROMPT`, `BOUNDED TRACE`, `multi-alias relay`, `TRACE LABEL CANDIDATE`가 제시되었다.
- 관련 회차: 제 26화
- 기존 BIBLE/outline과의 관계: 1권 outline의 공동 추적/`ORIGIN_NULL` 표시 비트를 원고화했다. 공동 프롬프트는 세 답을 합치지 않고 확인 가능/불가/거절 항목을 나누는 방식이다.
- 떡밥 상태 변화: `D-LAYER.SEAT09`는 단독 범인이 아니라 `multi-alias relay`의 출구 노드 가능성으로 남았다. `ORIGIN_NULL`은 라벨 후보일 뿐, 정체와 모델 묘지는 아직 설명하지 않는다.
- 후속 회차에서 조심할 점: 제 27화는 발표를 재개하되 `ORIGIN_NULL`을 범인으로 단정하지 않는다. 클레어의 음성 상실, 지아의 불확실성 표시, 미나의 피로와 징계 리스크를 유지한다.
- BIBLE 업데이트 제안 위치: 캐논 업데이트 대기열에 공동 프롬프트, 공개 경로 한정 추적, `multi-alias relay` 라벨 후보를 임시 캐논으로 검토.
